本文档记录实现新卡牌/器官/效果时需要了解的关键机制，基于代码实际调查结果。

## Effect 拦截机制

每个 ActionEvent 的每个 Effect 在执行前后各有一个触发周期。

源文件：`src/core/objects/game/transaction.ts`

```typescript
private async _applyEffects(event: ActionEvent) {
    for (let effect of event.effects) {
        await effect.trigger("before", 0)  // before 阶段：可以修改 effect.params
        await effect.apply()               // 执行效果函数
        await effect.trigger("after", 0)   // after 阶段：响应已发生的效果
    }
}
```

结论：
  useCard 事件中的 damage effect 执行前，`before make damage` 触发器会触发
  applyState effect 执行前，`before make applyState` 触发器会触发
  不需要显式 doEvent 包装，每个 effectUnit 的执行都有完整触发周期

## $triggerEffect.params(key) 语法

源文件：`src/core/utils/ReferenceResolver.ts`

```
$triggerEffect.params(key)   → 获取触发该反应的 Effect 的 params[key]
$triggerEffect.target.accessor(args)  → 获取触发 Effect 的事件目标的属性
$source.stateStack()         → 获取事件来源的状态层数
$owner.status(key)           → 获取 EffectUnit 所属器官/卡牌的 status
$battle.turn                 → 当前回合数
```

注意：语法是 `params(key)` 括号，不是 `params.key`

在 before 和 after 触发器中都可用。用于在反应系统的 effect 参数中读取触发它的 Effect 的值。

## targetType 选项

在 reaction 的事件定义中使用，由统一解析系统（TargetSpec.ts + resolveTriggerEventTarget）处理：

```
targetType: "triggerOwner"     → 触发该反应的实体（例如有状态的角色）
targetType: "triggerEffect"    → 触发该反应的 Effect 对象本身（用于修改 effect.params）
targetType: "triggerSource"    → 触发来源物品（器官/遗物自身）
targetType: "eventTarget"      → 原始事件的 target
targetType: "eventSource"      → 原始事件的 source
targetType: "owner"            → 拥有该物品（卡牌/器官/遗物）的角色
targetType: "self"             → 物品自身
targetType: { faction: "enemy" }  → 所有敌方存活单位
targetType: { faction: "player" } → 所有玩家存活单位
```

统一解析函数：`resolveTriggerEventTarget(targetType, triggerEvent, triggerEffect, item, owner)`
源文件：`src/core/objects/system/trigger/resolveTriggerEventTarget.ts`

reaction 系统、accumulateAndTrigger 均通过此函数解析 targetType，不要在新效果中手写 switch。

## triggerTarget（触发器挂载目标）

控制触发器挂载到哪个实体上（而非效果目标），用于器官/遗物 possess 交互中。

```
triggerTarget: "player"          → 玩家队伍第一个成员
triggerTarget: "opponent"        → 对手队伍第一个成员
triggerTarget: "allOpponents"    → 所有对手
triggerTarget: "allPlayers"      → 所有存活玩家
triggerTarget: "allEnemies"      → 所有存活敌人
triggerTarget: "randomOpponents" → 随机 count 个对手（需提供 count）
```

统一解析函数：`resolveTriggerMountTargets(spec, owner)`
源文件：`src/core/utils/resolveTriggerMountTargets.ts`

在 TriggerMap（ItemModifier / Organ）和 accumulateAndTrigger 中均通过此函数解析，
新增挂载目标类型只需修改这一处。

triggerTarget 与 targetType 的区别：
  triggerTarget — 触发器挂载位置（谁来响应事件）
  targetType    — 效果施加目标（谁承受效果）

## 卡牌 inHand 交互

卡牌有三种交互在不同时机触发：

possess
  触发时机：卡牌被添加到 CardModifier（通过器官获得、addCardsFromSource）
  触发事件 key："possessItem"
  使用场景：卡牌进入牌库时的一次性初始化

inHand
  触发时机：卡牌被移入手牌堆（drawCard、enterHand 调用时）
  触发事件 key："inHandCard"
  自动清理：卡牌离开手牌时 unmountInHand() 自动移除所有触发器
  使用场景：卡牌在手牌中的持续效果（如孢子牌：在手牌时回合结束加中毒）

use
  触发时机：卡牌被打出时

临时塞入 drawPile 的卡牌（如孢子牌）：
  possess 不触发（因为未经过 CardModifier）
  inHand 在抽到手牌时触发 ✅
  因此孢子牌的回合结束效果通过 inHand 交互实现

## 器官 possess 交互的 cleanup 机制

源文件：`src/core/objects/system/modifier/OrganModifier.ts`

loseOrgan() 调用流程：
  1. removeWorkTriggers() — 移除 work 触发器
  2. removeByItem(organ) — 触发 ItemModifierUnit.cleanup()
  3. ItemModifierUnit.cleanup() — 清理所有副作用（触发器、修饰器等）

结论：在 possess 的 triggers 中注册的触发器，失去器官时自动清理，不需要手动管理。

## StateModifier 公开 API

源文件：`src/core/objects/system/modifier/StateModifier.ts`

```typescript
stateModifier.addState(stateData, stacks, source)
stateModifier.removeState(stateKey, triggerRemoveEffect?)
stateModifier.changeStack(stateKey, stackKey, delta)
stateModifier.getState(stateKey): State | undefined
stateModifier.getAllStates(): State[]
stateModifier.hasState(stateKey): boolean
```

获取 state 的 category（用于判断是否为 debuff）：

```typescript
import { stateList } from "@/static/list/target/stateList"

const state = stateModifier.getState("poison")
// state.key 是状态的 key，在 stateList 中查找 category
const stateData = stateList.find(s => s.key === state.key)
const isDebuff = stateData?.category === "debuff"
```

或直接统计：
```typescript
const debuffCount = stateModifier.getAllStates()
    .filter(s => stateList.find(d => d.key === s.key)?.category === "debuff")
    .filter(s => {
        const stacks = s.stacks.find(st => st.key === "default")?.stack ?? 0
        return stacks > 0
    }).length
```

```typescript
const card = await getCardByKey("cardKey")
```

## 运行时修改 Status 值

enemy/player 的 status 字段是 Status 对象，有以下方法：

```typescript
// 直接设置基础值（会重新计算修饰器）
status.setOriginalBaseValue(newValue)

// 添加修饰器（可撤销）
const remover = status.addByJSON(source, {
    modifierValue: delta,
    targetLayer: "base",
    modifierType: "additive"
})
```

例子：在运行时改变 enemy 的 actions-per-turn：
```typescript
const actionsStatus = (enemy as any).status["actions-per-turn"]
if (actionsStatus && Number(actionsStatus.value) < 2) {
    actionsStatus.setOriginalBaseValue(2)
}
```

## Effect 拦截模板（剧毒心核：中毒+1）

```typescript
// 器官 possess 触发器：拦截 applyState effect
triggers: [{
    when: "before",
    how: "make",
    key: "applyState",
    action: "poisonAmplify"
}],
reaction: {
    poisonAmplify: [{
        key: "poisonAmplify",
        label: "剧毒心核：中毒+1",
        targetType: "triggerEffect",  // target 是 applyState Effect 对象
        effect: [{ key: "organ_poisonAmplify", params: {} }]
    }]
}

// EffectFunc
export const organ_poisonAmplify: EffectFunc = (event, _effect) => {
    const target = event.target
    if (Array.isArray(target) || !isEffect(target)) return false
    if (target.key !== "applyState") return false
    if (target.params.stateKey !== "poison") return false
    target.params.stacks = Number(target.params.stacks || 1) + 1
    return true
}
```

# 状态系统开发指南

状态（State）是附着在实体上的持续性效果，如中毒、力量、虚弱。本文档说明如何定义新状态、配置其行为，以及在效果中如何操作状态。

---

## 状态条目格式

状态定义在 `src/static/list/target/stateList.ts`，每条记录的完整字段：

```typescript
{
    label: string,               // 显示名称
    key: string,                 // 唯一标识符
    describe: Describe,          // 描述文本（字符串数组）
    category?: "buff" | "debuff" | "neutral",  // 分类，默认 neutral
    showType?: "number" | "bool",              // UI 显示方式，默认 number
    repeate?: "stack" | "refresh" | "none",    // 重复获得时的行为，默认 stack
    stacks?: Stack[] | number,                 // 初始堆叠结构，默认生成 default 为 0
    allowNegative?: boolean,                   // 是否允许负数层数，默认 false
    stackChange?: StackChangeRule[],           // 自动衰减规则
    interaction?: StateInteractionData,        // 状态自身的触发器行为
    checkExist?: (getter, state) => boolean,   // 自定义存在性检查（高级用法）
}
```

---

## 字段详解

### category

```
"buff"    — 正面状态
"debuff"  — 负面状态（影响"文物"状态抵挡逻辑、部分效果的判断）
"neutral" — 中性状态（默认）
```

### showType

```
"number"  — UI 上显示层数数字（默认）
"bool"    — UI 上只显示图标，不显示数字（如壁垒、文物等只需要有/无的状态）
```

### repeate

控制已有该状态时再次获得的行为：

```
"stack"   — 层数累加（最常用，如中毒、力量）
"refresh" — 重置为新值，不累加（如变硬）
"none"    — 忽略，不做任何改变
```

### stacks

简单状态直接省略此字段，系统自动创建一个 `key: "default"` 的堆叠：

```typescript
// 不填 stacks，applyState 时传数字即可
{ key: "applyState", params: { stateKey: "poison", stacks: 3 } }
```

需要多个维度时，手动定义 stacks 数组：

```typescript
stacks: [
    { key: "duration", stack: 0, label: "持续回合" },
    { key: "power",    stack: 0, label: "效果强度" }
]
```

多堆叠时，applyState 的 stacks 参数用数组形式：

```typescript
{ key: "applyState", params: {
    stateKey: "myState",
    stacks: [{ key: "duration", stack: 3 }, { key: "power", stack: 5 }]
}}
```

### stackChange（自动衰减）

定义状态在特定时机自动改变层数，不需要手写触发器：

```typescript
stackChange: [
    { timing: "turnEnd", delta: -1 },         // 回合结束时 default 层数 -1
    { timing: "turnStart", stackKey: "power", delta: -1 },  // 回合开始时 power 层 -1
    { timing: "turnEnd", delta: "all" },       // 回合结束时清零
]
```

timing 支持：`"turnStart"` / `"turnEnd"` / `"battleStart"` / `"battleEnd"`

stackChange 内部通过 `changeStateStack` 效果实现，会正常触发事件系统。

## 状态自带的触发器行为

状态可以通过 `interaction` 字段定义自己的触发逻辑，格式与器官的 possess 交互完全相同：

```typescript
interaction: {
    possess: {
        triggers: [{
            when: "before",
            how: "make",
            key: "damage",
            action: "powerBoost"
        }],
        reaction: {
            powerBoost: [{
                key: "powerBoost",
                label: "力量增伤",
                targetType: "triggerEffect",
                effect: [{ key: "modifyDamageValue", params: { delta: "$source.stateStack()" } }]
            }]
        }
    }
}
```

`$source.stateStack()` 是一个参数表达式，读取触发该反应的状态的 default 层数。读取指定堆叠用 `$source.stateStack('stackKey')`。

状态的 interaction 在状态附着到实体时注册触发器，移除时自动清理，与器官的 possess 机制一致。

## 典型模式

### 每回合造成伤害的 debuff（中毒类）

```typescript
{
    label: "中毒",
    key: "poison",
    category: "debuff",
    describe: ["回合结束时受到等量伤害"],
    showType: "number",
    repeate: "stack",
    stackChange: [{ timing: "turnEnd", delta: -1 }],
    interaction: {
        possess: {
            triggers: [{ when: "before", how: "take", key: "turnEnd", action: "poisonDamage" }],
            reaction: {
                poisonDamage: [{
                    key: "poisonDamage",
                    targetType: "triggerOwner",
                    effect: [{ key: "damage", params: { value: "$source.stateStack()" } }]
                }]
            }
        }
    }
}
```

### 增伤/减伤 buff（力量类）

```typescript
{
    label: "力量",
    key: "power",
    category: "buff",
    describe: ["造成的伤害增加"],
    showType: "number",
    repeate: "stack",
    allowNegative: true,
    interaction: {
        possess: {
            triggers: [{ when: "before", how: "make", key: "damage", action: "powerBoost" }],
            reaction: {
                powerBoost: [{
                    key: "powerBoost",
                    targetType: "triggerEffect",
                    effect: [{ key: "modifyDamageValue", params: { delta: "$source.stateStack()" } }]
                }]
            }
        }
    }
}
```

### 无层数的标记状态（bool 类）

```typescript
{
    label: "壁垒",
    key: "barricade",
    category: "buff",
    describe: ["护甲不在回合开始时消失"],
    showType: "bool",
    repeate: "stack",
    interaction: {
        possess: {
            triggers: [{
                when: "before",
                how: "take",
                key: "clearArmor",
                cancelEvent: true    // 直接取消事件，不需要 action+reaction
            }]
        }
    }
}
```

### 触发后消耗的一次性状态

```typescript
{
    label: "变硬",
    key: "harden",
    category: "buff",
    describe: ["受到攻击时减少伤害，触发后消耗"],
    showType: "number",
    repeate: "refresh",
    interaction: {
        possess: {
            triggers: [{
                when: "before",
                how: "take",
                key: "damage",
                action: "hardenAbsorb",
                requireFromAttackCard: true
            }],
            reaction: {
                hardenAbsorb: [
                    {
                        key: "reduce",
                        targetType: "triggerEffect",
                        effect: [{ key: "state_hardenAbsorb", params: { stacks: "$source.stateStack()" } }]
                    },
                    {
                        key: "remove",
                        targetType: "triggerOwner",
                        effect: [{ key: "removeState", params: { stateKey: "harden" } }]
                    }
                ]
            }
        }
    }
}
```

## 效果操作状态

### applyState — 施加状态

```typescript
{ key: "applyState", params: { stateKey: "poison", stacks: 2 } }

// 多堆叠
{ key: "applyState", params: {
    stateKey: "myState",
    stacks: [{ key: "duration", stack: 3 }, { key: "power", stack: 5 }]
}}
```

### removeState — 移除状态

```typescript
{ key: "removeState", params: { stateKey: "poison" } }

// 触发状态自身的 remove interaction
{ key: "removeState", params: { stateKey: "poison", triggerRemoveEffect: true } }
```

### changeStateStack — 修改层数

```typescript
{ key: "changeStateStack", params: { stateKey: "poison", delta: -2 } }

// 指定堆叠键
{ key: "changeStateStack", params: { stateKey: "myState", stackKey: "power", delta: 1 } }

// 取反（delta 符号反转）
{ key: "changeStateStack", params: { stateKey: "poison", delta: 1, negate: true } }
```

层数变化后若 `checkExist` 返回 false，状态自动移除。

## 新增状态的步骤

第一步：在 `stateList.ts` 添加状态定义
  选择合适的 category、showType、repeate
  用 stackChange 配置自动衰减（不要手写触发器做同样的事）
  用 interaction 定义状态自身的行为

第二步：若状态需要专属的 EffectFunc（如变硬的百分比减伤）
  在 `src/core/effects/` 下合适的目录创建效果
  命名用 `state_` 前缀，如 `state_hardenAbsorb`
  在 `effectMap.ts` 注册

第三步：验证
  applyState 施加后 checkExist 是否能正确判断存在性
  stackChange 是否在正确时机衰减
  失去来源（如器官被摧毁）时 interaction 触发器是否自动清理

# 游戏核心概念与设计理念

本文档说明 Born The Spire（蘇生尖塔）的游戏定位、核心循环和设计原则，供开发者和模组创作者了解项目的方向与初衷。

---

## 项目定位

Born The Spire 是一款以《杀戮尖塔》为灵感创作的中文 Roguelike 卡牌游戏，基于 Tauri 2 + Vue 3 构建，面向 Windows 桌面平台。

这是一个非盈利的粉丝项目，以教育和社区创作为目的，开源且欢迎社区参与。

---

## 核心游戏循环

```
进入战斗 → 打败敌人 → 从敌人身上选取1个器官 → 将器官组合到自己身上
                                ↓
                    器官提供新卡牌和被动能力
                                ↓
                    用组合后的体型进入下一场战斗
```

玩家不是在"收集卡牌"，而是在"吸收器官"。每个器官都是敌人真实拥有的一部分——玩家打败敌人后，可以选择将其某个器官同化到自己身上，获得该器官所携带的卡牌和被动效果。

这套循环决定了游戏的核心张力：
  打败哪种敌人，决定了能获得哪种器官
  选择哪个器官，决定了当前流派的走向
  器官之间如何组合，决定了最终的构建强度

---

## 器官系统

器官是游戏最核心的内容单元，同时承载三个功能：

卡牌来源
  器官持有期间，它携带的卡牌会加入玩家牌组
  失去器官时，这些卡牌自动从牌组中移除

被动效果
  器官通过触发器响应游戏事件，提供各种持续性被动
  例如：每次打出卡牌时额外抽一张，受到伤害时减免1点，回合开始时施加中毒

属性加成
  器官可以直接修改玩家的基础属性（最大生命、能量上限等）
  修改通过修饰器（Modifier）系统管理，失去器官时自动撤销

器官有"质量"这一概念，过度受到伤害可能导致器官损坏。损坏的器官失去主动功能，但可能触发新的被动行为（broken 交互）。

---

## 玩家可以对器官做什么

在器官奖励界面，玩家有多种选择：

同化（默认）
  将器官纳入自己身体，获得其卡牌和被动

吞噬（默认）
  消耗器官获取材料，用于后续升级

献祭、其他（需要解锁）
  特定遗物会解锁新的处理方式，提供不同的策略取向

这套选择机制是游戏策略深度的重要来源——不是每场战斗都应该同化器官。

## Mod 友好是设计出发点

整个项目从立项起就以支持模组（Mod）为核心设计目标，而不是功能完成后再考虑开放接口。

具体体现在：

内部与外部使用同一套 API
  游戏本体的卡牌、器官、遗物定义与模组的定义方式完全相同
  开发者不能写出"只有内部才能用"的代码

声明式优先于命令式
  内容通过 JSON 对象描述，而不是直接写逻辑代码
  触发器用 when/how/key + action/reaction 声明，不手写回调
  这使得没有编程经验的人也能创作内容

注册表模式
  所有内容类型（卡牌、器官、遗物、敌人、房间、事件）都通过注册表管理
  添加新内容只需往注册表里加条目，不需要修改核心逻辑

扩展性优先
  每个系统都设计为可以扩展，而不只是支持当前的使用场景
  例如器官奖励的处理方式、水池房间的行动，都可以通过遗物动态解锁新选项

## 内容设计原则

实现新内容时，应遵循以下优先级：

首先复用现有原子效果
  `damage` / `heal` / `applyState` / `gainArmor` 等通用效果覆盖大多数场景

其次考虑复合效果
  `accumulateAndTrigger` / `countAndTrigger` 等封装了常见模式

只在真正唯一的逻辑才写专属效果
  以 `organ_` 或 `card_` 前缀命名，明确标记不可复用

设计新机制时要问：模组作者能用这个吗？
  如果一个机制只有内部能用，说明接口设计需要改进

## 第一层内容方向

第一层的敌人围绕两大种族展开：

有机体
  以生物机能为主题，拥有中毒、寄生、增殖等机制
  器官偏向状态积累和被动触发

机械体
  以工程和能量为主题，拥有热量、充能、指挥等机制
  器官偏向数值爆发和连锁触发

两种种族的器官在同一套机制框架下运作，但提供截然不同的构建方向，玩家组合跨种族器官会产生新的化学反应。

---

## 与《杀戮尖塔》的关系

Born The Spire 以《杀戮尖塔》的 Roguelike 卡牌框架为灵感，但核心体验不同：

杀戮尖塔：选择卡牌构建牌组，遗物提供被动加成
Born The Spire：吸收器官构建身体，器官本身同时是卡牌来源和被动效果的载体

这套设计让"你打倒了什么"直接决定"你变成了什么"，身份认同和构建策略是同一件事。

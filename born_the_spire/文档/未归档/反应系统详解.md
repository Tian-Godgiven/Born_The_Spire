# 反应系统详解

反应系统（Reaction System）是器官、遗物、卡牌实现被动效果的核心机制。它通过声明式数据描述"当什么事件发生时，对谁做什么"，无需编写回调函数。

---

## 基本结构

每个器官/遗物/卡牌定义包含两个配合使用的字段：

```typescript
{
    interaction: {
        possess: {               // 交互阶段（possess / work / broken / lose 等）
            triggers: [{
                when: "before" | "after",
                how: "make" | "via" | "take",
                key: "eventKey",
                action: "myAction"   // 字符串键，指向 reaction 中的定义
            }]
        }
    },
    reaction: {
        myAction: [{             // 与 action 字段同名
            key: "eventKey",     // 要触发的事件类型
            label: "描述",
            targetType: "...",   // 效果作用在谁身上
            effect: [{ key: "...", params: {} }]
        }]
    }
}
```

`action` 是连接 triggers 和 reaction 的桥梁，字符串值任意命名，只需两侧一致。

---

## triggers 字段详解

### when / how / key

决定触发器响应哪类事件：

```
when: "before"  — 事件效果执行前触发（可修改效果参数）
when: "after"   — 事件效果执行后触发（响应已发生的变化）

how: "make"     — 持有者是事件的来源（造成事件）
how: "via"      — 持有者是事件的媒介（通过自身的事件）
how: "take"     — 持有者是事件的目标（被事件作用）

key: 事件类型，如 "damage" / "heal" / "turnStart" / "useCard" 等
```

完整的 key 列表见实现参考手册"已知 ActionEvent keys"一节。

### action

```typescript
triggers: [{
    when: "before",
    how: "take",
    key: "damage",
    action: "reduceDamage"   // 触发时，在 reaction.reduceDamage 中查找事件配置
}]
```

### triggerTarget（可选）

控制触发器挂载到哪个实体上，默认挂在 owner（持有者）上。

```typescript
triggers: [{
    when: "after",
    how: "take",
    key: "damage",
    action: "myAction",
    triggerTarget: { participantType: "entities", key: "allOpponents" }
}]
```

可用的 key 值：
```
"player"          — 随机1个存活玩家（语法糖）
"allOpponents"    — 所有对手
"allPlayers"      — 所有存活玩家
"allEnemies"      — 所有存活敌人
"randomOpponents" — 随机 count 个对手（需同时提供 count 字段）
不填              — 默认挂在 owner 上
```

triggerTarget 是"谁来监听事件"，与 targetType（"效果作用在谁身上"）是两个不同的概念，见下文。

### 其他可选字段

```typescript
{
    level?: number,              // 触发优先级，数值越大越先执行，默认 0
    condition?: Condition,       // 附加条件，条件不满足时不触发
    timing?: "immediate" | "battleStart",  // 注册时机，battleStart 表示战斗开始时才注册
    disableUntil?: "battleEnd",  // 触发一次后禁用直到战斗结束
    requirePositiveEffect?: boolean,  // 仅当事件包含正值效果时触发
    requireFromAttackCard?: boolean,  // 仅当伤害来自攻击牌时触发
    importantKey?: string,       // 关键触发器标识（用于管理和去重）
    onlyKey?: string,            // 唯一触发器标识（同 onlyKey 只保留最后一个）
}
```

## reaction 字段详解

reaction 是一个对象，键名与 action 字段对应，值为事件配置数组。

```typescript
reaction: {
    myAction: [                  // 可包含多个事件，依次执行
        {
            key: "damage",       // 要创建的事件类型
            label: "描述",       // 可选，用于调试日志
            targetType: "...",   // 效果作用在谁身上（必填）
            effect: [{ key: "...", params: {} }],  // 效果列表
            sourceTargetType?: "...",  // 事件 source（可选）
            mediumTargetType?: "...",  // 事件 medium（可选）
            condition?: Condition,    // 仅当满足条件时执行（可选）
        }
    ]
}
```

### targetType 选项

targetType 是"效果作用在谁身上"，由 resolveTriggerEventTarget 解析：

```
"owner"           — 物品的拥有者（最常用，器官/遗物的持有玩家或敌人）
"triggerOwner"    — 触发该反应的实体（通常与 triggerTarget 配合使用）
"triggerEffect"   — 触发该反应的 Effect 对象（用于修改效果参数，如减伤）
"eventTarget"     — 原始事件的目标
"eventSource"     — 原始事件的来源
"triggerSource"   — 触发器来源（通常是物品自身）
"allEnemies"      — 所有存活敌人
"allAllies"       — 所有玩家
"allEntities"     — 所有实体
"player"          — 玩家队伍第一个成员
"item"            — 物品自身
```

支持修饰符：
```
"allEnemies.random"  — 随机一个敌人
"allAllies.first"    — 第一个玩家
```

---

## triggerTarget 与 targetType 的区别

这是最容易混淆的两个字段。

```
triggerTarget  — 触发器挂载位置：谁来监听事件
targetType     — 效果施加目标：谁承受效果
```

典型场景（寄生菌根：对手每受到8点伤害，自己回血）：

```typescript
{
    interaction: {
        possess: {
            triggers: [{
                when: "after",
                how: "take",
                key: "damage",
                action: "lifeSteal",
                triggerTarget: { participantType: "entities", key: "allOpponents" }
                // 触发器挂在对手身上，当对手受到伤害时触发
            }]
        }
    },
    reaction: {
        lifeSteal: [{
            key: "heal",
            targetType: "triggerOwner",  // 效果作用在 owner（自己）上
            effect: [{ key: "heal", params: { value: 2 } }]
        }]
    }
}
```

- triggerTarget 决定"谁受伤时我感知到" → 对手
- targetType 决定"谁获得治疗" → 自己

## 交互阶段（interaction 的键名）

器官/遗物/卡牌支持不同的交互阶段，决定触发器的生命周期：

```
possess   — 器官持有期间持续生效，失去时自动清理所有注册的触发器
work      — 器官正常工作（未损坏）时生效，损坏后自动移除
broken    — 器官损坏后激活（替代 work 触发器）
get       — 获得时一次性触发（不注册触发器，只执行 effects）
lose      — 失去时一次性触发（不注册触发器，只执行 effects）
break     — 损坏时一次性触发
inHand    — （卡牌专用）在手牌中时生效，离开手牌自动清理
use       — （卡牌专用）打出时触发
```

possess 和 work 最常用。work 适合"器官正常发挥功能"的效果，破损后自动失效。

## 执行流程

以器官 possess 触发器为例：

```
1. acquireOrgan()
   → processPossessInteraction() 处理 possess 交互
   → processPossessTrigger() 解析 triggerTarget，确定挂载目标
   → mountPossessTrigger() 将触发器注册到目标实体

2. 事件发生（如 damage 事件执行）
   → transaction._applyEffects() 逐个执行 effect
   → 每个 effect 执行前后触发 effect.trigger("before/after", 0)
   → 匹配的触发器 callback 被调用

3. callback 执行（createTriggerByTriggerMap 中）
   → 在 source（器官/遗物）上查找 reaction[action]
   → 调用 executeItemReaction()
   → 解析 targetType，确定每个事件的目标
   → 调用 doEvent() 创建并执行新事件

4. loseOrgan() / breakOrgan()
   → removeWorkTriggers() 或 unit.cleanup()
   → 所有在 possess/work 中注册的触发器自动清理
```

## 常用模式

### 持有时的被动防御（减伤）

```typescript
// 器官：受伤时减少1点伤害
{
    interaction: {
        possess: {
            triggers: [{
                when: "before",     // before：在效果执行前拦截
                how: "take",        // 持有者承受伤害
                key: "damage",
                action: "reduceDamage"
            }]
        }
    },
    reaction: {
        reduceDamage: [{
            key: "reduceDamage",
            targetType: "triggerEffect",   // 目标是 damage Effect 对象本身
            effect: [{ key: "reduceDamageValue", params: { value: 1 } }]
        }]
    }
}
```

### 打牌时触发效果

```typescript
// 器官：打出一张牌后抽1张
{
    interaction: {
        work: {
            triggers: [{
                when: "after",
                how: "make",        // 持有者打出卡牌
                key: "useCard",
                action: "drawOnUse"
            }]
        }
    },
    reaction: {
        drawOnUse: [{
            key: "drawCard",
            targetType: "owner",
            effect: [{ key: "drawFromDrawPile", params: { value: 1 } }]
        }]
    }
}
```

### 回合开始/结束时触发

```typescript
// 器官：回合开始时对所有敌人施加1层中毒
{
    interaction: {
        work: {
            triggers: [{
                when: "after",
                how: "make",
                key: "turnStart",
                action: "poisonAll"
            }]
        }
    },
    reaction: {
        poisonAll: [{
            key: "applyState",
            targetType: "allEnemies",
            effect: [{ key: "applyState", params: { stateKey: "poison", stacks: 1 } }]
        }]
    }
}
```

### 一次性获得时效果

```typescript
// 器官：获得时立即回复5点生命（不注册持续触发器）
{
    interaction: {
        get: {
            target: { key: "owner" },
            effects: [{ key: "heal", params: { value: 5 } }]
        }
    }
}
```

### 单个 action 触发多个事件

```typescript
reaction: {
    myAction: [
        // 事件1：造成伤害
        {
            key: "damage",
            targetType: "eventTarget",
            effect: [{ key: "damage", params: { value: 3 } }]
        },
        // 事件2：同时获得护甲
        {
            key: "gainArmor",
            targetType: "owner",
            effect: [{ key: "gainArmor", params: { value: 2 } }]
        }
    ]
}
```

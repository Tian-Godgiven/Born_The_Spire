# 新内容开发清单

添加各类内容时需要操作的文件和最小格式。按需查阅对应章节。

---

## 效果（Effect）

每种内容都可能需要新效果，建议先确认效果再写内容。

需要修改的文件：
  `src/static/list/system/effectMap.ts` — 注册效果
  `src/core/effects/<分类>/` — 实现效果函数

effectMap 注册格式：

```typescript
import { myEffect } from "@/core/effects/category/myEffect"

{ label: "效果说明", key: "myEffect", effect: myEffect }
```

EffectFunc 格式：

```typescript
import type { EffectFunc } from "@/core/objects/system/effect/Effect"

export const myEffect: EffectFunc = (event, effect) => {
    const value = Number(effect.params.value ?? 0)
    // ... 实现逻辑
    return true
}
```

命名规范：
  原子效果 — 动词+名词，如 `gainArmor`、`drawCard`
  复合效果 — 动词短语，如 `accumulateAndTrigger`
  专属效果 — `organ_` 或 `card_` 前缀，如 `organ_heatTick`

## 状态（State）

需要修改的文件：
  `src/static/list/target/stateList.ts` — 添加状态定义

最小格式：

```typescript
{
    label: "中毒",
    key: "poison",
    describe: ["回合结束时受到等量伤害"],
    category: "debuff",
    showType: "number",
    repeate: "stack",
}
```

如需每回合衰减，加 `stackChange`：

```typescript
stackChange: [{ timing: "turnEnd", delta: -1 }]
```

如需状态自带触发行为，加 `interaction`（格式与器官 possess 完全相同）。

详细说明见《状态系统开发指南》。

## 卡牌（Card）

需要修改的文件：
  `src/static/list/item/cardList.ts` — 添加卡牌定义
  `src/static/list/system/effectMap.ts` — 如有新效果

最小格式（攻击牌）：

```typescript
{
    label: "打击",
    key: "original_card_00001",
    tags: ["attack"],
    status: { damage: 6, cost: 1 },
    describe: ["造成", { key: ["status", "damage"] }, "点伤害"],
    interaction: {
        use: {
            target: { faction: "opponent" },
            effects: [{ key: "damage", params: { value: 6 } }]
        }
    }
}
```

常用 tags：
  `"attack"` — 攻击牌（影响 requireFromAttackCard 触发器）
  `"skill"` — 技能牌
  `"power"` — 能力牌

卡牌词条（entry）：

```typescript
entry: ["card_void"]   // 虚无：回合结束自动消耗
entry: ["card_exhaust"] // 消耗：使用后移出战斗
```

若卡牌只通过器官给出，只需加到 cardList，再在器官的 `cards` 字段引用。若是普通可抽取卡牌，还需确认奖励/商店池配置（见游戏内容配置文档）。

## 器官（Organ）

需要修改的文件：
  `src/static/list/target/organList.ts` — 添加器官定义
  `src/static/list/item/cardList.ts` — 如有配套卡牌
  `src/static/list/system/effectMap.ts` — 如有专属效果

最小格式：

```typescript
{
    label: "石肤",
    key: "original_organ_00003",
    describe: ["受到的伤害-1"],
    rarity: OrganRarity.Uncommon,
    part: OrganPartEnum.Skin,
    status: { "max-mass": 25 },
    current: ["mass"],
}
```

带卡牌的器官：

```typescript
cards: ["original_card_00001"],   // 器官持有期间加入牌组
```

带被动效果的器官（possess 交互），格式见《反应系统详解》。

part 可选值（OrganPartEnum）：
  `Heart` / `Skin` / `Muscle` / `Gland` / `Core` / `Bone` / `Nerve`

rarity 可选值（OrganRarity）：
  `Common` / `Uncommon` / `Rare`

## 遗物（Relic）

需要修改的文件：
  `src/static/list/item/relicList.ts` — 添加遗物定义
  `src/static/list/system/effectMap.ts` — 如有新效果

最小格式：

```typescript
{
    label: "回血石",
    key: "original_relic_00001",
    describe: ["战斗结束时回复", { key: ["status", "heal"] }, "点生命"],
    pool: ["common"],
    status: { heal: 5 },
    interaction: {
        possess: {
            triggers: [{
                when: "after",
                how: "make",
                key: "battleEnd",
                action: "healOnBattleEnd"
            }]
        }
    },
    reaction: {
        healOnBattleEnd: [{
            key: "heal",
            targetType: "owner",
            effect: [{ key: "heal", params: { value: 5 } }]
        }]
    }
}
```

pool 常用值：
  `["common"]` — 普通奖励池
  `["shop"]` — 商店池
  `["boss"]` — Boss 奖励池

## 敌人（Enemy）

需要修改的文件：
  `src/static/list/target/enemyList.ts` — 添加敌人定义
  `src/static/list/target/organList.ts` — 如有专属器官
  `src/static/list/item/cardList.ts` — 如有专属卡牌

最小格式：

```typescript
{
    label: "史莱姆",
    key: "enemy_slime",
    status: { "max-health": 40 },
    organ: ["organ_slime_body"],   // 敌人拥有的器官（提供卡牌和被动）
    behavior: {
        patterns: [],
        fallback: {
            intent: "attack",
            action: {
                selector: { tags: ["attack"] },
                mode: "random"
            },
            describe: "随机使用攻击牌"
        }
    }
}
```

behavior 说明：
  `patterns` — 条件行为列表，按 priority 从高到低匹配，第一个满足条件的生效
  `fallback` — 无任何 pattern 满足时的默认行为

pattern 格式：

```typescript
{
    priority: 10,
    intent: "attack",
    condition: { /* 见敌人AI文档 */ },
    action: {
        selector: { tags: ["attack"] },
        mode: "random"   // random / loop / weighted
    },
    describe: "条件说明"
}
```

## 战斗（Battle）

需要修改的文件：
  `src/static/list/room/battle/battleList.ts` — 添加战斗配置
  （确保引用的敌人已在 enemyList 中注册）

最小格式：

```typescript
{
    key: "battle_f1_slime",
    name: "史莱姆巢穴",
    battleType: "normal",   // normal / elite / boss
    enemyConfigs: [
        { key: "enemy_slime" }
    ]
}
```

多敌人：

```typescript
enemyConfigs: [
    { key: "enemy_slime" },
    { key: "enemy_slime" },   // 同类型可以重复
    { key: "enemy_mage" }
]
```

## 检查项汇总

添加任何内容后确认：

  key 是否全局唯一（与现有内容不冲突）
  effectMap 是否已注册所有用到的效果 key
  organList 是否已注册器官引用的所有 part/rarity 枚举值（不要写字符串字面量）
  新效果是否处理了 event.target 为数组的情况（使用 handleEventEntity 包裹）
  专属效果文件是否只在 effectMap 中被引用，没有被其他内容直接 import

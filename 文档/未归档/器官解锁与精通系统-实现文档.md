# 器官解锁与精通系统 - 实现文档

完整的元进度系统，包含器官解锁、精通记录、精英击败奖励等功能

## 系统概述

这是一个跨局持久化的元进度系统，玩家通过游戏过程解锁器官、积累精通、击败精英来提升初始器官选择能力。

核心特性
  持久化存储：使用 LocalStorage，支持导出/导入
  器官解锁：同化器官时自动解锁
  器官精通：通关时标记当前持有器官的精通等级
  精英奖励：击败新精英类型增加初始器官费用上限
  开始页面：选择初始器官、设置种子、选择进阶等级

## 数据结构

### MetaProgressSaveV1

```typescript
interface MetaProgressSaveV1 {
    version: 1

    // 已解锁的器官 key 集合
    unlockedOrgans: Record<string, true>

    // 器官进阶精通记录
    // key: organKey
    // value: 该器官达到过的最高进阶等级
    organAscensionMastery: Record<string, number>

    // 已击败的精英敌人类型 key 集合
    defeatedEliteTypes: Record<string, true>
}
```

### 器官费用配置

```typescript
ORGAN_RARITY_COST = {
    common: 1,      // 普通
    uncommon: 2,    // 稀有
    rare: 3,        // 传说
    unique: 4       // 独特
}

INITIAL_ORGAN_BUDGET_BASE = 4  // 基础上限
INITIAL_ORGAN_BUDGET_MAX = 7   // 最大上限
```

费用上限计算
  base + 击败精英种类数，不超过 max
  例如：击败 2 种精英 → 上限 = min(4 + 2, 7) = 6

## 数据流程

### 器官解锁流程

```
战斗胜利 → 器官奖励 → 玩家选择"同化"
  ↓
assimilateAction (organRewardActions.ts)
  ↓
modifier.assimilateOrgan(organ)
  ↓
解锁逻辑：
  loadMetaProgress()
  unlockOrgan(metaProgress, organ.key)
  saveMetaProgress(metaProgress)
```

### 器官精通流程

```
游戏通关（无下一层级）
  ↓
BattleRoom.triggerFloorSelection()
  ↓
processVictoryMastery(nowPlayer)
  ↓
遍历玩家持有的器官（未损坏）
  ↓
markOrganAscensionMastery(metaProgress, organKey, ascensionLevel)
  ↓
saveMetaProgress(metaProgress)
```

### 精英击败流程

```
精英战斗胜利
  ↓
BattleRoom.complete() - player_win 分支
  ↓
检查 battleType === "elite" || "elitePlus"
  ↓
遍历所有敌人
  ↓
processEliteDefeat(enemy.key)
  ↓
markEliteDefeated(metaProgress, eliteKey)
  ↓
首次击败 → 上限 +1
  ↓
saveMetaProgress(metaProgress)
```

### 开始游戏流程

```
Start.vue 加载
  ↓
loadMetaProgress() → 获取已解锁器官列表
  ↓
从 organList 创建 Organ 实例
  ↓
显示器官网格（费用、精通标记）
  ↓
玩家选择器官（验证费用）
  ↓
点击"开始游戏"
  ↓
startNewRun(seed, ascensionLevel, selectedOrgans)
  ↓
使用 gainOrgan 效果添加初始器官
  ↓
进入游戏
```

## 使用方式

### 玩家视角

开始游戏
  1. 进入开始页面
  2. 左侧设置种子（可选）和进阶等级
  3. 右侧选择初始器官（受费用限制）
  4. 精通的器官显示金色边框和"精通"标记
  5. 右键点击器官查看详情
  6. 点击"开始游戏"

游戏过程
  1. 战斗胜利后获得器官奖励
  2. 选择"同化"器官 → 自动解锁
  3. 击败精英敌人 → 增加初始器官上限（首次）
  4. 游戏通关 → 标记当前持有器官的精通等级

存档管理
  1. 点击"导出存档" → 复制到剪贴板
  2. 点击"导入存档" → 从剪贴板读取并导入

### 开发者视角

添加新器官
  1. 在 organList 中定义器官数据
  2. 设置 rarity 字段（决定费用）
  3. 玩家同化后自动解锁

添加新精英
  1. 在 enemyList 中定义敌人数据
  2. 创建 BattleRoom 时设置 battleType: "elite"
  3. 击败后自动记录

调试命令
  ```javascript
  // 查看当前元进度
  const { loadMetaProgress } = await import('@/core/persistence/metaProgress')
  console.log(loadMetaProgress())

  // 手动解锁器官
  const { loadMetaProgress, saveMetaProgress, unlockOrgan } = await import('@/core/persistence/metaProgress')
  const data = loadMetaProgress()
  unlockOrgan(data, 'organ_heart_001')
  saveMetaProgress(data)

  // 清空进度
  const { clearMetaProgress } = await import('@/core/persistence/metaProgress')
  clearMetaProgress()
  ```

## 注意事项

数据一致性
  器官解锁在同化时触发，不在战斗结束时
  器官精通在游戏通关时触发，不在战斗结束时
  精英击败在战斗胜利时触发

精通判断逻辑
  记录最高精通等级，不是等级列表
  显示精通：masteryLevel >= currentAscension
  例如：器官在进阶 5 精通，当前选择进阶 3 → 显示精通

费用限制
  开始游戏时必须验证总费用不超过上限
  UI 实时显示当前费用和上限
  超出上限时禁用"开始游戏"按钮

器官损坏
  战斗结束时，损坏的器官不计入精通
  检查 organ.isDisabled 字段

存档版本
  当前版本：V1
  未来如需迁移，在 loadMetaProgress 中处理版本转换

LocalStorage 限制
  大小限制：通常 5-10MB
  当前数据量很小，不会有问题
  如果未来数据量增大，考虑压缩或使用 IndexedDB

初始器官不触发解锁
  startNewRun 中使用 gainOrgan 直接添加
  不会触发解锁逻辑（因为已经是解锁状态）

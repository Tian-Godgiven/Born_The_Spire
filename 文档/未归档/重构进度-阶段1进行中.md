# 阶段 1 进度报告：类型与值分离

## 目标

将所有 `import { Entity }` 改为 `import type { IEntity }`，实现类型与值的分离，避免循环依赖。

## 已完成的工作

### 1. 修改核心工具文件

#### typeGuards.ts ✓
- 将 `Entity`, `Effect` 改为使用 `IEntity`, `IEffect` 接口
- 所有类型守卫函数现在返回接口类型而不是具体类
- 保留对 `State` 和 `Card` 的类型引用（待后续阶段处理）

```typescript
// 之前
import type { Entity } from "@/core/objects/system/Entity"
export function isEntity(participant: EventParticipant): participant is Entity

// 之后
import type { IEntity } from "@/core/interfaces"
export function isEntity(participant: EventParticipant): participant is IEntity
```

### 2. 修改核心系统文件

#### Trigger.ts ✓
- `initTriggerByMap` 参数改为 `IEntity`
- `onTrigger` 参数改为 `IActionEvent`, `IEffect`
- `resolveTriggerEventTarget` 返回类型改为 `IEntity | IEntity[] | IEffect`
- 移除 `instanceof Entity` 检查，改用 `participantType` 属性检查

#### Effect.ts ✓
- `EffectConstructor.triggerEvent` 改为 `IActionEvent`
- `Effect.actionEvent` 改为 `IActionEvent`
- `apply` 方法参数改为 `Partial<IActionEvent>`
- 导入改为使用接口

#### EffectFunc.ts ✓
- `resolveEffectParams` 参数改为 `IActionEvent`, `IEffect`
- 导入改为使用接口

## 当前编译错误

### 主要问题

很多效果函数（damage.ts, heal.ts, gainItem.ts 等）仍然使用 `Entity` 类型，导致类型不兼容：

```
error TS2345: Argument of type 'IEntity' is not assignable to parameter of type 'Entity'.
```

### 原因分析

这些文件中的函数接受 `Entity` 参数，但现在 typeGuards 返回 `IEntity`。需要修改这些函数的签名。

## 策略调整建议

当前的"自下而上"策略（先修改底层工具，再修改上层使用者）遇到了大量类型不兼容问题。

建议调整为"自上而下"策略：

1. **先让具体类实现接口**
   ```typescript
   export class Entity implements IEntity {
       // 现有代码不变
   }
   ```

2. **然后逐步修改使用者**
   - 函数参数从 `Entity` 改为 `IEntity`
   - 变量声明从 `Entity` 改为 `IEntity`

3. **最后移除对具体类的依赖**
   - 所有 import 改为 import type
   - 使用工厂函数创建对象

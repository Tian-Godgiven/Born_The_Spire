# 动画系统

全局动画基础设施，基于 GSAP，统一管理所有 UI 动画的注册、播放、冲突和回调。

## 基本信息

位置：`src/ui/animation/`
底层：GSAP（项目已有依赖）
入口：`src/ui/animation/index.ts`

### 文件结构

```
src/ui/animation/
  index.ts              统一导出
  types.ts              类型定义
  AnimationManager.ts   全局管理器单例
  useAnimation.ts       组件内 composable
  presets.ts            内置预设动画
```

## 核心概念

### 效果模式（mode）

overlay
  保持原 DOM 不变，叠加视觉效果
  例：闪烁、发光、变色、震动

replace
  临时替换 DOM 内容播放动画
  例：死亡演出、变身动画
  通过 replaceComponent 指定替换组件

append
  在 DOM 旁边/上方添加额外元素
  例：跳字、粒子效果
  通过 appendComponent 指定附加组件

### Channel 系统

同 channel 的动画互斥，不同 channel 并行
  channel: "color" → 闪红和闪绿互斥
  channel: "position" → 震动和后退互斥
  不同 channel 之间自由叠加

### 优先级（priority）

同 channel 内，高优先级动画可以打断低优先级
  默认 priority: 0
  死亡动画 priority: 999，打断一切

### 重复策略（repeat）

同 key 动画重复触发时的行为
  restart：重新开始（默认）
  ignore：忽略新触发
  queue：排队等当前播完

### 可打断（interruptible）

默认 true，允许被同 channel 更高优先级动画打断
设为 false 时，其他动画只能排队等待

## 动画定义

### 声明式简写（简单动画）

```typescript
{
  key: "death_fadeout",
  mode: "overlay",
  channel: "death",
  priority: 999,
  animate: {
    to: { opacity: 0 },
    duration: 0.6,
    ease: "power1.in"
  }
}
```

animate 字段支持：
  from + to：fromTo 动画
  仅 from：from 动画
  仅 to：to 动画
  duration：持续时间（秒），默认 0.5
  ease：缓动函数，默认 "power1.inOut"

### build 函数（复杂动画）

```typescript
{
  key: "hit_shake",
  mode: "overlay",
  channel: "position",
  build: (el) => {
    return gsap.timeline({ paused: true })
      .to(el, { x: -6, duration: 0.04 })
      .to(el, { x: 6, duration: 0.04 })
      .to(el, { x: -4, duration: 0.04 })
      .to(el, { x: 4, duration: 0.04 })
      .to(el, { x: 0, duration: 0.04 })
  }
}
```

build 接收 DOM 元素和可选的 params 参数，返回 paused 的 GSAP Timeline
build 和 animate 二选一，build 优先

### 时间轴回调

在动画定义中声明回调节点，调用方决定具体行为：

```typescript
// 定义
{
  key: "boss_death",
  mode: "overlay",
  channel: "death",
  build: (el) => { /* ... */ },
  callbacks: [
    { at: 300, action: "dropLoot" },
    { at: 500, action: "playSound" }
  ]
}

// 使用
await play("boss_death", {
  onAction: (actionKey, elapsed) => {
    if (actionKey === "dropLoot") spawnLoot()
    if (actionKey === "playSound") playDeathSound()
  }
})
```

at：触发时机（毫秒）
action：回调标识 key

## 使用方式

### 组件内使用（useAnimation）

```vue
<template>
  <div ref="animRef">
    <!-- replace 模式组件 -->
    <component v-if="replaceComponent" :is="replaceComponent" />
    <slot v-else />
    <!-- append 模式附加项 -->
    <component v-for="item in appendItems" :key="item.id"
      :is="item.component" v-bind="item.props" />
  </div>
</template>

<script setup>
import { useAnimation } from "@/ui/animation"

const { animRef, play, cancel, cancelAll, isAnimating, replaceComponent, appendItems } = useAnimation("my_element_id")

// 播放动画，返回 handle
const handle = await play("hit_flash")

// 等待动画完成
await handle.promise

// 带回调播放
await play("boss_death", {
  params: { intensity: 2 },
  onAction: (key, ms) => { /* ... */ },
  onStart: () => { /* ... */ },
  onEnd: () => { /* ... */ },
  onCancel: () => { /* ... */ }
})

// 取消动画
cancel("color")      // 取消指定 channel
cancelAll()           // 取消所有（含队列）
```

useAnimation 返回值：
  animRef：绑定到 DOM 元素的 template ref
  play(animKey, options?)：播放动画，返回 Promise<AnimationHandle>
  cancel(channel?)：取消指定 channel 或所有 channel
  cancelAll()：取消所有动画和队列
  isAnimating：是否有活跃动画（computed）
  replaceComponent：replace 模式当前组件（computed）
  appendItems：append 模式附加项列表（computed）

### AnimationHandle

```typescript
const handle = await play("death_fadeout")

handle.key          // 动画 key
handle.channel      // channel
handle.promise      // Promise<void>，动画结束时 resolve
handle.cancel()     // 主动取消
handle.isPlaying()  // 是否正在播放
```

### 直接使用 AnimationManager

```typescript
import { animationManager } from "@/ui/animation"

// 注册
animationManager.register({ key: "custom", mode: "overlay", /* ... */ })

// 播放（需要先 bind 元素）
animationManager.bind("enemy_1", domElement)
await animationManager.play("enemy_1", "hit_flash")

// 全局取消（场景切换时）
animationManager.cancelGlobal()
```

## 注册动画

### 注册预设动画

在 main.ts 中自动注册：

```typescript
import("@/ui/animation").then(({ registerPresetAnimations }) => {
  registerPresetAnimations()
})
```

### Mod 注册自定义动画

```typescript
import { animationManager } from "@/ui/animation"

animationManager.register({
  key: "boss_explode",
  mode: "replace",
  channel: "death",
  priority: 999,
  interruptible: false,
  replaceComponent: BossExplodeAnimation,
  callbacks: [
    { at: 500, action: "screenShake" },
    { at: 1000, action: "dropReward" }
  ]
})
```

## 内置预设

death_fadeout
  渐隐，channel: death，priority: 999，不可打断
  用于实体死亡

hit_flash
  白闪，channel: color
  用于受击反馈

hit_shake
  左右震动，channel: position
  用于受击反馈

heal_glow
  绿光，channel: color
  用于治疗反馈

fade_in / fade_out
  淡入/淡出，channel: visibility

scale_in
  缩放弹入，channel: visibility

pulse
  脉冲放大缩小，channel: emphasis

## CharaAnimator 组件

位置：`src/ui/components/object/Target/Chara/components/CharaAnimator.vue`

基于动画系统的角色动画包装组件，自动监听 isAlive 播放死亡动画。

```vue
<CharaAnimator ref="animator" :target="enemy" death-animation="boss_explode">
  <div>角色内容</div>
</CharaAnimator>
```

Props：
  target（Chara）：角色实体
  deathAnimation（string）：死亡动画 key，默认 "death_fadeout"

暴露：
  isDying（ref<boolean>）：是否正在播放死亡动画
  isDead（ref<boolean>）：是否已死亡（动画播完）
  play：转发 useAnimation 的 play 方法

死亡后使用 visibility: hidden 保留占位，不影响其他元素布局。

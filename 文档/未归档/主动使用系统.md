# 主动使用系统

玩家可以主动触发器官和遗物的特殊能力的系统

## 实现状态

✅ 已完成核心实现
  核心系统类已实现
  UI组件已实现
  生命周期集成已完成
  测试示例已创建

## 设计原则

为mod制作者提供灵活且易用的主动能力系统
  制式方法：提供常见场景的便捷配置
  回调接口：允许完全自定义的复杂交互
  组合设计：多种限制和触发方式可以自由组合

## 核心分类

### A. 如何使用（触发方式）

定义玩家如何触发能力

左键行为
  固定为查看详情面板
  不可配置

右键行为
  直接触发：右键直接执行能力（usage.type: "instant"）
  弹出菜单：右键显示能力选择菜单（usage.type: "menu"）
  选择目标：右键后进入目标选择模式（usage.type: "targetSelection"）
  开关切换：右键切换激活/关闭状态（usage.type: "toggle"）

### B. 使用限制（可组合）

定义能力的使用限制，多个限制可以同时存在

消耗型限制（costs）
  energy：能量消耗
  health：生命消耗
  material：材料消耗
  可以同时消耗多种资源

次数型限制（usesPerTurn, usesPerBattle, usesPerFloor, usesTotal）
  每回合可用次数
  每战斗可用次数
  每层级可用次数
  总可用次数（永久消耗）

冷却型限制（cooldown）
  使用后冷却X回合
  冷却期间不可使用

充能型限制（charges）
  maxCharges：最大充能数
  startingCharges：初始充能数
  required：使用需要的充能数
  gainOn：如何获得充能
    turnStart：回合开始
    turnEnd：回合结束
    battleStart：战斗开始
    battleEnd：战斗结束
    cardPlayed：打出卡牌时
    damageTaken：受到伤害时

条件型限制（conditions）
  healthBelow：生命值低于X%
  healthAbove：生命值高于X%
  minHandSize：最少手牌数
  maxHandSize：最多手牌数
  hasStatus：拥有某状态
  notHasStatus：不拥有某状态
  scene：使用场景（combat, nonCombat, anytime）

自定义检查（customCheck）
  完全自定义的可用性检查函数
  返回 { available: true } 表示可用
  返回 { available: false, reason: string } 表示不可用
  函数签名：(item, ability, owner) => { available: boolean, reason?: string }

### C. 视觉反馈

定义物品的视觉状态显示

角标系统（badges）
  position：显示位置（topRight, topLeft, bottomRight, bottomLeft）
  type：角标类型（text, icon, number）
  getValue：获取显示值的函数
  getColor：获取颜色的函数
  showWhen：显示条件函数

整体状态
  右键点击时自动显示能力菜单或执行能力
  有主动能力的物品显示特殊标识（⚡图标）
  不可用时显示置灰效果

## 数据结构

### 实际实现的结构

```typescript
// 在 Relic 或 Organ 的配置中
{
  label: string
  key: string
  describe: Describe

  // 主动能力列表
  activeAbilities?: Array<{
    key: string
    label: string
    description?: string

    // 使用方式
    usage: {
      type: "instant" | "menu" | "targetSelection" | "toggle"

      // 菜单配置（当 type 为 "menu" 时）
      menuConfig?: {
        title?: string
        items: Array<{
          type: "ability" | "callback"
          abilityKey?: string
          label?: string
          description?: string
          callback?: Function
        }>
      }

      // 目标选择配置（当 type 为 "targetSelection" 时）
      targetConfig?: {
        type: "enemy" | "ally" | "card" | "organ"
        count?: {
          min?: number
          max?: number
        }
        filter?: (target: Entity) => boolean
      }
    }

    // 消耗
    costs?: {
      energy?: number
      health?: number
      material?: number
    }

    // 使用限制
    restrictions?: {
      usesPerTurn?: number
      usesPerBattle?: number
      usesPerFloor?: number
      usesTotal?: number
      cooldown?: number

      charges?: {
        maxCharges: number
        startingCharges?: number
        required: number
        gainOn?: {
          event: "turnStart" | "turnEnd" | "battleStart" | "battleEnd" | "cardPlayed" | "damageTaken"
          amount: number
        }
      }

      conditions?: {
        healthBelow?: number
        healthAbove?: number
        minHandSize?: number
        maxHandSize?: number
        hasStatus?: string
        notHasStatus?: string
        scene?: "combat" | "nonCombat" | "anytime"
      }

      customCheck?: (item: Entity, ability: ActiveAbility, owner: Entity) => {
        available: boolean
        reason?: string
      }
    }

    // 效果定义
    effects?: EffectUnit[]

    // 开关型能力的特殊配置
    onToggleOn?: EffectUnit[]
    onToggleOff?: EffectUnit[]

    // 触发器配置（用于持续效果）
    triggers?: TriggerMap

    // 自定义执行逻辑（高级）
    onActivate?: (item: Entity, owner: Entity, context: any) => Promise<void>
  }>

  // 显示配置
  display?: {
    badges?: Array<{
      position: "topRight" | "topLeft" | "bottomRight" | "bottomLeft"
      type: "text" | "icon" | "number"
      getValue: (item: Entity, owner: Entity) => string | number
      getColor?: (item: Entity, owner: Entity) => string
      showWhen?: (item: Entity, owner: Entity) => boolean
    }>
  }
}
```

## 实现架构

### 核心类

ActiveAbilityManager (src/core/objects/system/ActiveAbilityManager.ts)
  主协调器，处理右键事件和能力执行
  handleRightClick(item, owner, abilities)：处理右键点击
  executeAbility(item, ability, owner)：执行能力
  executeInstantAbility(item, ability, owner)：执行即时能力
  executeMenuAbility(item, ability, owner)：显示菜单并执行选择
  executeTargetSelectionAbility(item, ability, owner)：进入目标选择模式
  executeToggleAbility(item, ability, owner)：切换开关状态

RestrictionChecker (src/core/objects/system/RestrictionChecker.ts)
  检查各种使用限制
  checkAbilityUsability(item, ability, owner)：综合检查可用性
  checkCosts(ability, owner)：检查消耗
  checkUsageRestrictions(item, ability)：检查次数限制
  checkCooldown(item, ability)：检查冷却
  checkConditions(ability, owner)：检查条件
  checkCharges(item, ability)：检查充能
  checkCustom(item, ability, owner)：执行自定义检查

UsageTracker (src/core/objects/system/UsageTracker.ts)
  追踪能力使用状态
  recordUsage(item, abilityKey)：记录使用
  applyCooldown(item, abilityKey, turns)：应用冷却
  addCharge(item, abilityKey, amount)：添加充能
  consumeCharges(item, abilityKey, amount)：消耗充能
  setToggleState(item, abilityKey, active)：设置开关状态
  getUsageState(item, abilityKey)：获取使用状态

DisplayManager (src/core/objects/system/DisplayManager.ts)
  管理视觉反馈
  calculateBadges(item, owner, abilities)：计算角标
  shouldDisable(item, ability, owner)：检查是否置灰
  getHighlight(item, ability, owner)：获取高亮状态

ActiveAbilityLifecycleManager (src/core/objects/system/ActiveAbilityLifecycleManager.ts)
  管理生命周期事件
  handleTurnStart(item, abilities)：处理回合开始
  handleTurnEnd(item, abilities)：处理回合结束
  handleBattleStart(item, abilities)：处理战斗开始
  handleBattleEnd(item, abilities)：处理战斗结束

### 生命周期集成

回合开始时 (src/core/effects/turn.ts)
  重置 usesPerTurn 次数限制
  减少冷却计数
  触发 gainOn: "turnStart" 的充能
  调用 handleTurnStart() 处理所有主动能力

回合结束时 (src/core/effects/turn.ts)
  调用 handleTurnEnd() 处理所有主动能力

战斗开始时 (src/core/objects/target/Player.ts)
  重置 usesPerBattle 次数限制
  重置冷却
  调用 handleBattleStart() 处理所有主动能力

战斗结束时 (src/core/objects/game/battle.ts)
  清理战斗相关状态
  调用 handleBattleEnd() 处理所有主动能力

## 注意事项

限制检查顺序
  先检查条件限制（最快失败）
  再检查次数和冷却
  再检查充能
  最后检查消耗（避免误扣资源）

消耗的应用时机
  检查通过后才扣除消耗
  如果执行失败，应该退还消耗
  消耗在 RestrictionChecker 中检查，在 ActiveAbilityManager 中扣除

开关型能力的状态管理
  状态存储在 UsageTracker 中
  通过 getToggleState() 和 setToggleState() 管理
  关闭时需要移除相关效果（通过 onToggleOff）

充能系统的实现
  充能数存储在 UsageTracker 中
  通过生命周期事件触发充能获取
  使用时消耗充能，不足时无法使用

自定义回调的错误处理
  回调函数应该捕获异常
  避免崩溃影响游戏
  使用 try-catch 包裹自定义逻辑

性能优化
  可用性检查可能频繁调用
  考虑缓存检查结果
  只在状态变化时重新计算

生命周期集成
  主动能力系统已集成到回合和战斗生命周期
  在 Player.startBattle() 中处理战斗开始
  在 Battle.endBattle() 中处理战斗结束
  在 turn.ts 中处理回合开始/结束

UI交互
  右键点击自动触发主动能力
  有主动能力的物品显示⚡标识
  不可用时显示置灰效果和原因
  菜单和目标选择使用独立组件

## Mod制作指南

推荐流程
  先使用制式方法实现基础功能
  如果制式方法不够用，再使用自定义回调
  复杂交互可以完全使用 onActivate 回调

常见模式
  简单主动技能：usage.type: "instant" + effects
  多技能选择：usage.type: "menu" + menuConfig
  条件触发：使用 restrictions.conditions 限制
  资源消耗：使用 costs 定义
  冷却管理：使用 cooldown 和角标显示
  充能系统：使用 charges 配置

调试技巧
  使用 customCheck 返回 { available: false, reason: string } 来显示不可用原因
  使用 console.log 在回调中输出调试信息
  在开发者控制台使用 listActiveAbilities() 查看当前能力
  使用测试命令快速添加测试物品

扩展性考虑
  预留足够的自定义空间
  不要过度依赖制式方法
  复杂逻辑优先使用 onActivate 回调
  可以通过 customCheck 实现任意复杂的条件判断

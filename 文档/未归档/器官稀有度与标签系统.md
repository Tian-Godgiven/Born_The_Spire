# 器官稀有度与标签系统

## 稀有度系统 Quality

器官的稀有度反映其强度和稀有程度，影响多个游戏系统。

### 稀有度等级

  Common（普通）
    颜色：白色 #FFFFFF
    吞噬获取：20 物质
    黑市价格：100 金币

  Uncommon（罕见）
    颜色：绿色 #1EFF00
    吞噬获取：35 物质
    黑市价格：180 金币

  Rare（稀有）
    颜色：蓝色 #0070DD
    吞噬获取：55 物质
    黑市价格：280 金币

### 稀有度影响的系统

  吞噬收益
    吞噬器官获得的物质量 = 吞噬获取量 × 等级 + 稀有度加成
    稀有度越高，吞噬获得的物质越多

  升级成本
    升级消耗 = 吞噬获取量 × 110% ± 10% 波动
    稀有度越高，升级消耗越多

  修复成本
    修复消耗 = 吞噬获取量 × 60%
    稀有度越高，修复消耗越多

  同化成本
    同化消耗 = 吞噬获取量 × 20%
    稀有度越高，同化消耗越多

  黑市价格
    购买价格 = 基础价格 × (1 + (等级 - 1) × 20%)
    稀有度越高，黑市价格越贵

  出售价格
    出售收益根据稀有度和等级计算
    出售器官比出售等量物质收益高 20%

### 使用方式

在器官配置中指定稀有度：

```typescript
{
    label: "石芯",
    key: "original_organ_00002",
    quality: OrganQuality.Uncommon,  // 指定稀有度
    part: OrganPart.Heart,
    // ...
}
```

自定义吞噬获取量（可选）：

```typescript
{
    label: "特殊器官",
    quality: OrganQuality.Common,
    absorbValue: 50,  // 覆盖默认的 20
    // ...
}
```

## 标签系统 Tags

标签用于标识特殊类型的器官，不影响数值，主要用于过滤和显示。

### 预定义标签

  cursed（诅咒）
    颜色：深红色 #8B0000
    说明：诅咒器官无法正常移除，可能带来强大效果但伴随严重副作用

  cancer（癌变）
    颜色：深紫色 #4B0082
    说明：癌变器官可能扩散到其他器官，损坏时有特殊惩罚

  starter（初始）
    颜色：灰色 #808080
    说明：初始器官无法移除，是角色的核心组成部分

  unique（唯一）
    颜色：金色 #FFD700
    说明：唯一器官不能同时拥有多个

  temporary（临时）
    颜色：天蓝色 #87CEEB
    说明：临时器官在战斗结束后消失

  fragile（脆弱）
    颜色：橙色 #FFA500
    说明：脆弱器官更容易损坏

  indestructible（不朽）
    颜色：青色 #00CED1
    说明：不可摧毁的器官永远不会损坏

  synthetic（合成）
    颜色：紫色 #9370DB
    说明：人工合成的器官，可能有特殊机制

  mutated（变异）
    颜色：绿色 #32CD32
    说明：变异器官具有不稳定的效果

### 使用方式

在器官配置中添加标签：

```typescript
{
    label: "癌变心脏",
    key: "example_organ_cursed_001",
    quality: OrganQuality.Common,
    tags: [OrganTags.CURSED, OrganTags.CANCER],  // 添加标签
    // ...
}
```

使用自定义标签：

```typescript
{
    label: "特殊器官",
    quality: OrganQuality.Rare,
    tags: ["custom_tag", "another_tag"],  // 可以使用任意字符串
    // ...
}
```

### 工具函数

检查标签：

```typescript
import { hasTag, hasAnyTag, hasAllTags } from "@/static/list/target/organTags"

// 检查是否有指定标签
if (hasTag(organ.tags, OrganTags.CURSED)) {
    // 诅咒器官的特殊处理
}

// 检查是否有任意一个标签
if (hasAnyTag(organ.tags, OrganTags.CURSED, OrganTags.CANCER)) {
    // 有诅咒或癌变标签
}

// 检查是否有所有标签
if (hasAllTags(organ.tags, OrganTags.CURSED, OrganTags.CANCER)) {
    // 同时有诅咒和癌变标签
}
```

获取标签配置：

```typescript
import { getTagConfig, getTagLabel, getTagColor } from "@/static/list/target/organTags"

// 获取标签配置
const config = getTagConfig(OrganTags.CURSED)
// { key: "cursed", label: "诅咒", color: "#8B0000", description: "..." }

// 获取显示名称
const label = getTagLabel(OrganTags.CURSED)  // "诅咒"

// 获取颜色
const color = getTagColor(OrganTags.CURSED)  // "#8B0000"
```

### 标签的作用

标签是纯标记，不自动触发逻辑。各个系统根据标签实现特定行为：

  黑市过滤
    黑市可以通过标签过滤器官
    例如：普通黑市不出售诅咒器官，诅咒黑市只出售诅咒器官

  移除限制
    系统检查 cursed 标签，阻止正常移除
    需要特殊方式（如净化）才能移除

  战斗结束清理
    系统检查 temporary 标签，战斗结束时自动移除

  唯一性检查
    系统检查 unique 标签，阻止同时拥有多个

  UI 显示
    根据标签显示特殊图标或颜色
    鼠标悬停显示标签说明

### 添加新标签

在 `organTags.ts` 中添加新标签：

```typescript
export const OrganTags = {
    // ... 现有标签
    MY_NEW_TAG: "my_new_tag"
} as const

export const tagConfigList: TagConfig[] = [
    // ... 现有配置
    {
        key: OrganTags.MY_NEW_TAG,
        label: "新标签",
        color: "#FF0000",
        description: "这是一个新标签"
    }
]
```

## 设计原则

  稀有度管"强度"和"稀有程度"
    影响数值（物质、金钱、成本）
    影响出现概率

  标签管"特殊机制"和"获取限制"
    不影响数值
    用于过滤和特殊逻辑

  两个系统独立
    一个器官可以同时有稀有度和多个标签
    稀有度是必填的，标签是可选的

  数值后期调整
    当前数值仅供参考
    重点是机制的实现
    数值平衡在后期进行

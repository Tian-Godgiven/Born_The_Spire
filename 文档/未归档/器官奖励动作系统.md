# 器官奖励动作系统

器官奖励动作系统允许玩家在战斗奖励中选择器官时，对器官执行不同的动作（同化、吞噬、献祭等）。系统支持通过遗物动态解锁新动作。

## 核心概念

器官奖励动作 (OrganRewardAction)
  定义：对战斗奖励中的器官可以执行的操作
  内置动作：同化（获得器官）、吞噬（获得物质）
  扩展动作：献祭（需要遗物解锁）等

动作状态管理
  存储位置：Player.enabledOrganRewardActions (Set<string>)
  启用方式：通过 effect 添加到玩家的动作集合
  禁用方式：通过 effect 从玩家的动作集合移除

动作注册表
  全局注册表：organRewardActionRegistry
  存储所有可用的动作定义和处理器
  通过 enabled 函数动态判断动作是否对玩家可用

## 系统架构

### 类型定义

文件：`src/core/types/organRewardAction.ts`

```typescript
export interface OrganRewardAction {
  key: string           // 唯一标识
  label: string         // 显示名称
  icon: string          // 图标
  description: string   // 描述文本
  priority: number      // 排序优先级（越高越靠前）
  enabled: boolean | ((organ: Organ, player: Player) => boolean)
}

export interface OrganRewardActionContext {
  source: "battleReward" | "blackStore" | "event"
  battleType?: "normal" | "elite" | "boss"
}

export type OrganRewardActionHandler = (
  organ: Organ,
  player: Player,
  context: OrganRewardActionContext
) => Promise<boolean> | boolean
```

### 注册表

文件：`src/static/registry/organRewardActionRegistry.ts`

主要方法：
  registerAction(config, handler) - 注册新动作
  getAvailableActions(organ, player, context) - 获取玩家可用的动作列表
  executeAction(key, organ, player, context) - 执行指定动作
  unregisterAction(key) - 移除动作（一般不使用）

工作流程：
  1. 应用启动时注册所有内置动作
  2. 获取可用动作时，遍历注册表并检查 enabled 条件
  3. enabled 函数检查 player.enabledOrganRewardActions 集合
  4. 返回按 priority 排序的可用动作列表

### 内置动作

文件：`src/core/hooks/organRewardActions.ts`

同化动作 (assimilate)
  优先级：100
  效果：调用 OrganModifier.assimilateOrgan()
  可用条件：始终可用 (enabled: true)

吞噬动作 (devour)
  优先级：90
  效果：调用 OrganModifier.devourOrgan()
  可用条件：始终可用 (enabled: true)

献祭动作 (sacrifice)
  优先级：80
  效果：回复 10 点生命值
  可用条件：player.enabledOrganRewardActions.has("sacrifice")
  解锁方式：获得血祭之石遗物

初始化：
  initOrganRewardActions() 在应用启动时调用
  注册所有内置动作到全局注册表

### 效果函数

文件：`src/core/effects/organReward/organRewardActionEffects.ts`

启用动作效果 (enableOrganRewardAction)
  参数：actionKey (string)
  效果：player.enabledOrganRewardActions.add(actionKey)
  用途：遗物 possess 时启用新动作

禁用动作效果 (disableOrganRewardAction)
  参数：actionKey (string)
  效果：player.enabledOrganRewardActions.delete(actionKey)
  用途：遗物 lose 时禁用动作

注册到 effectMap：
  key: "enableOrganRewardAction"
  key: "disableOrganRewardAction"

## UI 交互

### 器官选择配置

文件：`src/ui/hooks/interaction/organChoice.ts`

```typescript
export interface OrganChoiceConfig {
  organKeys: string[]
  minSelect?: number
  maxSelect?: number
  cancelable?: boolean
  title?: string
  description?: string
  actionMode?: "selectOnly" | "selectThenAction"
  availableActions?: string[]
}

export interface OrganChoiceResult {
  selectedKeys: string[]
  selectedActions?: Map<string, string>  // organKey -> actionKey
}
```

actionMode 说明：
  selectOnly - 只选择器官，不选择动作（默认）
  selectThenAction - 选择器官后再选择动作

### 器官选择组件

文件：`src/ui/components/interaction/OrganChoice.vue`

交互流程：
  1. 玩家点击器官卡片 → 器官被选中并高亮
  2. 显示动作按钮组（同化、吞噬、献祭等）
  3. 玩家点击动作按钮 → 立即执行并关闭弹窗
  4. 玩家可点击"返回"按钮取消选择

关键状态：
  selectedOrgans: string[] - 选中的器官 key 列表
  organActions: Map<string, string> - 每个器官选择的动作
  availableActions - 从注册表获取的可用动作列表

关键函数：
  toggleSelection(organKey) - 选择/取消选择器官
  selectAction(organKey, actionKey) - 选择动作并立即执行
  deselectOrgan(organKey) - 取消选择器官（返回按钮）

### 奖励页面集成

文件：`src/ui/components/interaction/RewardPage.vue`

```typescript
async function openOrganChoice(reward: any) {
  const result = await openOrganRewardChoice({
    title: '选择器官',
    description: reward.getDisplayDescription(),
    organKeys: reward.organOptions.map((organ: any) => organ.key),
    minSelect: reward.selectCount,
    maxSelect: reward.selectCount,
    cancelable: true,
    actionMode: 'selectThenAction'
  })

  if (result.selectedKeys.length === 0) return

  reward.selectedOrgans = result.selectedKeys
  reward.selectedActions = result.selectedActions ?? new Map()
  await reward.claim()
}
```

## 遗物扩展机制

### 血祭之石示例

文件：`src/static/list/item/relicList.ts`

```typescript
{
  label: "血祭之石",
  describe: ["战斗奖励中，器官选择时可以选择献祭，回复生命值"],
  key: "original_relic_sacrifice",
  interaction: {
    possess: {
      target: { key: "owner" },
      effects: [{
        key: "enableOrganRewardAction",
        params: { actionKey: "sacrifice" }
      }]
    },
    lose: {
      target: { key: "owner" },
      effects: [{
        key: "disableOrganRewardAction",
        params: { actionKey: "sacrifice" }
      }]
    }
  }
}
```

工作流程：
  获得遗物 → possess 交互触发 → enableOrganRewardAction effect
  → player.enabledOrganRewardActions.add("sacrifice")
  → 献祭选项出现在器官奖励中

  失去遗物 → lose 交互触发 → disableOrganRewardAction effect
  → player.enabledOrganRewardActions.delete("sacrifice")
  → 献祭选项消失

### 添加新动作的步骤

1. 在 organRewardActions.ts 中定义动作处理器

```typescript
export const researchAction: OrganRewardActionHandler = async (organ, player) => {
  // 自定义逻辑：研究器官获得知识点
  const knowledgeGain = 5
  // ... 实现逻辑
  return true
}
```

2. 在 initOrganRewardActions() 中注册动作

```typescript
organRewardActionRegistry.registerAction({
  key: "research",
  label: "研究",
  icon: "📚",
  description: "研究器官获得知识点",
  priority: 70,
  enabled: (organ, player) => {
    return player.enabledOrganRewardActions.has("research")
  }
}, researchAction)
```

3. 创建解锁该动作的遗物

```typescript

    possess: {
      target: { key: "owner" },
      effects: [{
        key: "enableOrganRewardAction",
        params: { actionKey: "research" }
      }]
    },
    lose: {
      target: { key: "owner" },
      effects: [{
        key: "disableOrganRewardAction",
        params: { actionKey: "research" }
      }]
    }
  }
}
```

## 设计优势

状态明确
  玩家的可用动作存储在 Player 对象中
  不依赖全局状态或遗物查询

自动清理
  失去遗物时自动禁用动作
  通过 lose 交互实现，无需手动管理

数据驱动
  遗物配置完全通过 TriggerMap
  无需编写代码，只需配置 JSON

易于扩展
  添加新动作只需三步：定义处理器、注册动作、创建遗物
  Mod 开发者可以轻松添加自定义动作

架构一致
  与其他系统（StateModifier、ItemModifier）保持一致
  使用相同的 effect 系统和 TriggerMap 机制

## 注意事项

动作优先级
  priority 越高，在 UI 中越靠前显示
  建议：内置动作 100-90，扩展动作 80 以下

enabled 函数性能
  每次获取可用动作时都会调用 enabled 函数
  保持函数简单，避免复杂计算

多选模式限制
  当前实现针对单选模式（selectCount = 1）优化
  多选模式下，选择动作后立即执行可能不合适
  未来可能需要调整为"选择所有器官和动作后再确认"

动作执行失败
  executeAction 返回 boolean 表示成功/失败
  失败时会记录错误日志，但不会阻止其他器官的处理

遗物失效 vs 失去
  disableRelic 只是暂时失效，不触发 lose 交互
  loseRelic 才会触发 lose 交互并禁用动作
  注意区分这两种情况

## 架构说明

### 依赖方向（单向）

```
types (最底层，无依赖)
  ↓
interfaces (只依赖 types)
  ↓
base (依赖 interfaces 和 types)
  ↓
implementations (依赖 base, interfaces, types)
  ↓
factories (依赖 interfaces，创建 implementations)
  ↓
container (最上层，管理所有服务)
```

### 关键原则

1. **类型与值分离**：types 和 interfaces 只包含类型定义，不包含实现
2. **依赖接口不依赖实现**：所有代码应该依赖 IEntity 而不是 Entity 类
3. **工厂创建对象**：使用 createEntity() 而不是 new Entity()
4. **容器管理依赖**：通过 container.resolve() 获取服务

# 效果预览系统

效果预览系统允许在不真正执行效果的情况下，提前计算效果的最终数值。主要用于UI显示，例如在卡牌上显示考虑了所有增益/减益后的最终伤害值。

## 核心概念

预览系统通过模拟事件执行流程，但不修改实际游戏状态，来计算效果的最终值。

工作流程：
  创建模拟事件（MockEvent）
  执行触发器（会生成子事件，如 modifyDamage）
  处理生成的子事件（递归执行）
  调用效果的 preview 方法获取最终值

## 核心文件

逻辑：`src/core/utils/effectPreview.ts`
UI使用：`src/ui/components/object/Card.vue`
效果定义：`src/static/list/system/effectMap.ts`

## 主要API

### previewCardEffects

预览卡牌的所有效果

```typescript
function previewCardEffects(
    card: Card,
    source: Entity,
    target?: Entity | Entity[]
): PreviewResult
```

参数：
  card：要预览的卡牌
  source：使用卡牌的实体（通常是玩家）
  target：目标实体（可选）

返回：PreviewResult 对象，键为效果类型，值为最终数值

示例：

```typescript
import { previewCardEffects } from '@/core/utils/effectPreview'

const result = previewCardEffects(card, nowPlayer, enemy)
// result = { damage: 12, heal: 5 }
```

### previewEffectValue

预览单个效果的值（便捷方法）

```typescript
function previewEffectValue(
    card: Card,
    source: Entity,
    effectKey: string,
    target?: Entity | Entity[]
): number
```

参数：
  card：要预览的卡牌
  source：使用卡牌的实体
  effectKey：效果类型（如 "damage"）
  target：目标实体（可选）

返回：该效果的最终数值

示例：

```typescript
import { previewEffectValue } from '@/core/utils/effectPreview'

const finalDamage = previewEffectValue(card, nowPlayer, "damage", enemy)
// finalDamage = 12
```

## 模拟事件（MockEvent）

预览系统使用模拟事件来代替真实的 ActionEvent

结构：

```typescript
interface MockEvent {
    key: string                    // 事件类型
    source: Entity                 // 事件来源
    target: Entity | Entity[] | null  // 事件目标
    effects: EffectUnit[]          // 效果列表
    isPreview: true                // 标记为预览模式
    spawnedEvents: MockEvent[]     // 捕获的子事件
    spawnEvent: (config: any) => void  // 生成子事件的方法
}
```

特点：
  isPreview 标记为 true，效果函数可以据此判断是否为预览模式
  spawnEvent 会捕获子事件而不是真正执行
  不会修改实际游戏状态

## 为效果添加预览支持

在 effectMap 中为效果定义添加 preview 方法

### 基础效果预览

对于简单效果，直接返回效果值

```typescript
{
    label: "造成伤害",
    key: "damage",
    effect: damageTo,
    preview: (event, effect) => {
        return Number(effect.params.value)
    }
}
```

### 修改类效果预览

对于修改其他效果的效果，需要修改目标效果的 params.value

```typescript
{
    label: "修改伤害值",
    key: "modifyDamageValue",
    effect: modifyDamageValue,
    preview: (event, effect) => {
        const delta = Number(effect.params.delta)
        const targetEffect = event.effects[0]
        if (targetEffect && targetEffect.key === "damage") {
            const oldValue = Number(targetEffect.params.value)
            targetEffect.params.value = oldValue + delta
        }
        return null  // 修改类效果不返回值
    }
}
```

### 百分比修改效果预览

```typescript
{
    label: "按百分比修改伤害值",
    key: "modifyDamageByPercent",
    effect: modifyDamageByPercent,
    preview: (event, effect) => {
        const percent = Number(effect.params.percent)
        const targetEffect = event.effects[0]
        if (targetEffect && targetEffect.key === "damage") {
            const oldValue = Number(targetEffect.params.value)
            const delta = Math.round(oldValue * percent)
            targetEffect.params.value = oldValue + delta
        }
        return null
    }
}
```

## 预览方法签名

```typescript
type PreviewFunc = (event: MockEvent, effect: EffectUnit) => number | null
```

参数：
  event：模拟事件对象
  effect：当前效果单元

返回值：
  number：效果的最终数值（会被收集到 PreviewResult 中）
  null：不返回数值（通常用于修改类效果）

## 触发器执行

预览系统会执行触发器来模拟完整的事件流程

执行的触发器：
  source 的 before_make 触发器
  target 的 before_take 触发器

触发器可以生成子事件（如 modifyDamage 事件），这些子事件会被捕获并递归处理

示例流程：

```
1. 玩家使用攻击卡牌（damage: 8）
2. 触发器检测到 before_make_damage
3. 玩家有 +50% 伤害的 buff
4. 生成 modifyDamage 子事件（modifyDamageByPercent: 0.5）
5. 子事件修改原始伤害：8 * 0.5 = 4，最终伤害 = 12
6. preview 返回 { damage: 12 }
```

## UI集成示例

在 Card.vue 中的使用

```typescript
import { previewCardEffects } from '@/core/utils/effectPreview'

// 计算预览结果
const previewResult = computed(() => {
    return previewCardEffects(card, nowPlayer, hoverTarget)
})

// 创建增强的卡牌对象，用 Proxy 拦截 status.value 访问
const enhancedCard = computed(() => {
    return new Proxy(card, {
        get(target, prop) {
            if (prop === 'status') {
                return new Proxy(target.status, {
                    get(statusTarget, statusProp) {
                        const originalValue = statusTarget[statusProp]
                        if (originalValue && 'value' in originalValue) {
                            return new Proxy(originalValue, {
                                get(statusObj, valueProp) {
                                    if (valueProp === 'value') {
                                        const effectType = statusProp
                                        // 使用预览值
                                        if (previewResult.value[effectType] !== undefined) {
                                            return previewResult.value[effectType]
                                        }
                                    }
                                    return statusObj[valueProp]
                                }
                            })
                        }
                        return originalValue
                    }
                })
            }
            return target[prop]
        }
    })
})

// 使用增强的卡牌对象渲染描述
const describe = computed(() => {
    return getDescribe(card.describe, enhancedCard.value)
})
```

工作原理：
  previewResult 计算所有效果的最终值
  enhancedCard 使用 Proxy 拦截对 card.status.xxx.value 的访问
  当描述系统读取 card.status.damage.value 时，返回预览计算的值
  UI 自动显示最终伤害，无需修改描述模板

## 注意事项

preview 方法不应修改实际游戏状态
  只修改 MockEvent 和 EffectUnit 的临时数据
  不要修改 Entity 的属性

修改类效果应返回 null
  只有最终效果（如 damage、heal）返回数值
  修改类效果（如 modifyDamageValue）修改目标效果的 params 后返回 null

触发器中的效果函数应检查 isPreview
  如果效果函数需要区分预览和实际执行，检查 event.isPreview
  预览模式下跳过副作用（如日志、动画等）

子事件会被递归处理
  预览系统会自动处理触发器生成的所有子事件
  确保触发器在预览模式下也能正确生成子事件

## 扩展性

添加新效果的预览支持：
  在 effectMap 中为效果添加 preview 方法
  如果是修改类效果，修改目标效果的 params.value
  如果是最终效果，返回计算后的数值

支持新的效果类型：
  系统自动支持任何在 effectMap 中定义了 preview 的效果
  不需要修改 effectPreview.ts 核心代码

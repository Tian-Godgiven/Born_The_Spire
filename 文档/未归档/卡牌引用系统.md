# 卡牌引用系统

卡牌引用系统允许在describe中标记卡牌，使其可以在UI中hover显示完整的卡牌信息。

## 占位符格式

### {@:索引或ID} - 卡牌实例引用

用于引用器官提供的卡牌实例。

静态数据中使用索引：

```typescript
{
    label:"战斗之心",
    describe:["提供1张",{@:0},"卡牌到牌组"],
    cards:["original_card_00001"]
}
```

运行时自动替换为实例ID：

```typescript
// 器官获得后，describe自动更新为：
describe: ["提供1张",{@:"card_instance_123"},"卡牌到牌组"]
```

### {#:卡牌key} - 卡牌key预览

用于临时效果或预览，不引用实际实例。

```typescript
{
    describe:["下回合获得1张",{#:"original_card_00001"}]
}
```

## 工作流程

### 静态数据阶段

器官数据中使用索引标记卡牌：

```typescript
{
    label:"狂暴腺体",
    describe:["提供1张",{@:0},"和1张",{@:1},"到牌组"],
    cards:["original_card_00002", "original_card_00004"]
    // {@:0} 对应 cards[0] - "消耗打击"
    // {@:1} 对应 cards[1] - "肌肉强化"
}
```

### 实例化阶段

器官被获得时（OrganModifier.acquireOrgan）：

```typescript
// 1. 创建卡牌实例并添加到牌堆
const addedCards = await cardModifier.addCardsFromSource(organ, organ.cards)

// 2. 更新describe，将索引替换为实例ID
organ.describe = organ.describe.map(segment => {
    if (typeof segment === 'object' && '@' in segment && typeof segment['@'] === 'number') {
        const index = segment['@']
        return { '@': addedCards[index].__id }
    }
    return segment
})
```

### 渲染阶段

UI组件（OrganDetail.vue）渲染时：

```typescript
// 1. 解析describe，获取结构化数据
const segments = getDescribeStructured(organ.describe, organ)

// 2. 查找卡牌实例，更新显示文本
segments.map(segment => {
    if (segment.type === 'card') {
        const card = findCardById(segment.cardRef)  // 从牌堆查找
        return { ...segment, text: `【${card.label}】` }
    }
})

// 3. 渲染时，卡牌名称可hover显示Card组件
```

## Describe类型定义

```typescript
export type Describe = (
    string
    | {key:string[]}           // 访问属性
    | {$:string}               // 术语标记
    | {@:string|number}        // 卡牌实例引用（ID或索引）
    | {#:string}               // 卡牌key预览
)[]
```

## DescribeSegment类型

```typescript
export type DescribeSegment = {
    text: string
    type: 'plain' | 'value' | 'glossary' | 'card'
    cardRef?: string | number          // 卡牌引用（ID、key或索引）
    cardRefType?: 'instance' | 'key'   // 引用类型
    // ...
}
```

## 使用场景

### 器官提供的卡牌

使用 {@:索引}，实例化时自动替换为ID：

```typescript
{
    describe:["提供1张",{@:0},"到牌组"],
    cards:["original_card_00001"]
}
```

### 临时效果预览

使用 {#:key}，创建临时实例显示：

```typescript
{
    describe:["战斗开始时获得1张",{#:"original_card_00001"}]
}
```

### 多张卡牌

使用多个索引：

```typescript
{
    describe:["提供",{@:0},"、",{@:1},"和",{@:2}],
    cards:["card_key_1", "card_key_2", "card_key_3"]
}
```

## 卡牌查找逻辑

### 实例引用 {@:id}

```typescript
// 从玩家牌堆中查找
function findCardById(cardId: string): Card | null {
    const cardModifier = getCardModifier(nowPlayer)
    const allCards = cardModifier.getAllCards()
    return allCards.find(card => card.__id === cardId) || null
}
```

### Key预览 {#:key}

```typescript
// 创建临时实例
function createCardFromKey(cardKey: string): Card | null {
    const { cardList } = require('@/static/list/item/cardList')
    const cardData = cardList.find(c => c.key === cardKey)
    if (!cardData) return null
    return new Card(cardData)
}
```

## 注意事项

索引在运行时不应出现
  静态数据中的索引应该在器官获得时被替换为ID
  如果运行时遇到索引，会输出警告

卡牌失去时的处理
  当卡牌因事件失去时，describe中的引用会失效
  UI渲染时会找不到卡牌，显示为空

临时卡牌的生命周期
  使用 {#:key} 创建的临时卡牌仅用于显示
  不会添加到牌堆，不会影响游戏状态

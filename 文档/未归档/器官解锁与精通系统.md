# 器官解锁与精通系统

开始页面的初始器官选择功能，包含器官解锁、精通记录、精英击败奖励等元进度系统

## 核心需求

### 1. 元进度持久化

使用 LocalStorage 存储跨局数据
  已解锁的器官列表
  器官精通记录（每个器官达到过的最高进阶等级）
  已击败的精英敌人类型
  支持导出/导入存档（通过剪贴板）

### 2. 器官解锁机制

战斗结束时，如果玩家仍然持有某个器官（未损坏），则：
  如果该器官未解锁，则解锁
  标记该器官在当前进阶等级精通

精通显示：
  器官在开始页面显示时，如果其精通等级 >= 当前选择的进阶等级，则显示"精通"标记
  精通是纯展示系统，不影响游戏玩法（未来可能加入成就）

### 3. 初始器官费用系统

器官稀有度对应费用：
  common（普通）: 1
  uncommon（稀有）: 2
  rare（传说）: 3
  unique（独特）: 4

初始费用上限：
  基础上限：4
  最大上限：7
  每击败一种新的精英敌人，上限 +1（不超过最大上限）

### 4. 开始页面 UI 布局

左侧：
  上方：种子输入按钮（点击弹出种子输入弹窗）
  下方：进阶选择器（左右按钮切换，显示进阶值和介绍）

右侧：
  器官网格列表
    显示所有已解锁的器官
    显示器官名称
    点击器官显示详情弹窗（与游戏内器官 hover 弹窗一致）
    可以左右切换器官
    显示器官是否精通（根据当前选择的进阶等级）
    显示器官费用和当前已选总费用/上限

底部：
  开始游戏按钮
  导出存档按钮

## 数据结构

### MetaProgressSaveV1

```typescript
interface MetaProgressSaveV1 {
    version: 1

    // 已解锁的器官 key 集合
    unlockedOrgans: Record<string, true>

    // 器官进阶精通记录
    // key: organKey
    // value: 该器官达到过的最高进阶等级
    organAscensionMastery: Record<string, number>

    // 已击败的精英敌人类型 key 集合
    defeatedEliteTypes: Record<string, true>
}
```

### OrganUnlockData（旧版，待迁移）

```typescript
interface OrganUnlockData {
    unlockedOrgans: string[]
    organMastery: Record<string, {
        ascensionLevels: number[]
    }>
    organBudgetMax: number
    defeatedElites: string[]
}
```

注意：
  当前代码中存在两套数据结构
  MetaProgressSaveV1 是新版设计（使用 Record<string, true> 和最高等级记录）
  OrganUnlockData 是旧版设计（使用数组和等级列表）
  organUnlock.ts 目前使用的是 OrganUnlockData
  需要统一为 MetaProgressSaveV1

## 设计决策

### 为什么使用 Record<string, true> 而不是 string[]

性能：O(1) 查找 vs O(n) 查找
  存储空间：差异不大（JSON 序列化后 Record 稍大）
  代码简洁：`data.unlockedOrgans[key]` vs `data.unlockedOrgans.includes(key)`

### 为什么记录最高精通等级而不是等级列表

简化逻辑：只需要比较 masteryLevel >= currentAscension
  存储空间：更小
  未来扩展：如果需要详细记录，可以在 V2 中添加

### 为什么使用 LocalStorage 而不是文件系统

跨平台：Web 和 Tauri 都支持
  简单：无需处理文件路径和权限
  轻量：数据量小，不需要复杂的存储方案

### 为什么通过剪贴板导出而不是文件下载

简单：无需处理文件保存对话框
  跨平台：剪贴板 API 在 Web 和 Tauri 都可用
  用户友好：可以直接粘贴到聊天工具分享存档

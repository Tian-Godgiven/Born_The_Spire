# 过程事件 ActionEvent

过程事件是游戏进程中发生的某一件事，以及这件事产生的效果的过程。

过程事件是一个带有记录性质的对象，其本身并不直接影响游戏进程（造成影响的始终是效果）。

## 作用

统一管理事件的触发与日志
  记录事件的来源、媒介、目标和效果
  提供事件的执行流程控制

触发对象的触发器
  是【唯一一个】调用触发器的接口
  在事件执行前后触发参与实体的触发器

通过队列机制管理事件的执行顺序
  事件收集到事务Transaction中
  按顺序执行，支持事件嵌套和级联

## 结构

```typescript
{
    key: string                    // 事件key
    uuId: string                   // 唯一识别码
    source: EventParticipant       // 执行该事件的来源
    medium: EventParticipant       // 参与事件的媒介
    target: EventParticipant|EventParticipant[]  // 接受该事件的目标
    info: Record<string,any>       // 该事件执行全程的信息
    effects: Effect[]              // 该事件将会产生的效果
    transaction?: Transaction      // 所属事务
    simulate: boolean              // 是否为模拟事件（不实际执行效果）
    onExecute?: (event) => void    // 事件执行时的回调
    onComplete?: (event) => void   // 事件执行完成后的回调
}
```

### 参与者角色

source: 事件的发起者
  造成伤害的敌人
  使用卡牌的玩家
  触发效果的遗物

medium: 事件的媒介
  被使用的卡牌
  发动的攻击
  触发的能力

target: 事件的目标
  受到伤害的玩家
  被治疗的角色
  抽卡的玩家

## 执行流程

过程事件采用队列机制，完整流程如下：

### 1. 创建事件 (doEvent)

创建 ActionEvent 实例
  存储事件相关的各种信息（source, medium, target, effects）
  分配唯一识别码 uuId
  根据 effectUnits 创建 Effect 对象数组

### 2. 收集到队列

根据上下文决定收集方式：

在 before/after 触发器中
  添加到 eventCollector（立即递归执行）
  保证触发器响应的同步性

在效果函数中
  插入到当前事务的指定位置（top/bottom/index）
  默认插入队首（优先执行）

没有当前事务
  创建新事务并加入事务队列
  自动开始处理

### 3. 事务处理 (Transaction.process)

按顺序从队列中取出事件
  依次执行每个事件
  处理事件级联和嵌套

### 4. 执行单个事件 (Transaction.executeEvent)

Event before 触发器
  触发 source.makeEvent(before, key)
  触发 medium.viaEvent(before, key)
  触发 target.takeEvent(before, key)
  收集触发器产生的新事件到 eventCollector
  递归执行这些新事件

执行每个 Effect
  Effect before 触发器 → 递归执行新事件
  Effect.apply() 应用效果
  Effect after 触发器 → 递归执行新事件

Event after 触发器
  触发 source.makeEvent(after, key)
  触发 medium.viaEvent(after, key)
  触发 target.takeEvent(after, key)
  收集触发器产生的新事件到 eventCollector
  递归执行这些新事件

## 过程事件与触发器

过程事件在执行时会触发参与实体的触发器，触发时机分为：

Event before
  事件执行前触发
  可以修改事件参数（如伤害值）
  可以阻止事件执行

Effect before
  每个效果执行前触发
  可以修改效果参数
  可以添加额外效果

Effect after
  每个效果执行后触发
  用于响应效果结果
  可以触发联动效果

Event after
  事件执行后触发
  用于响应整个事件
  可以触发后续事件

### 触发器产生的新事件处理

在 before/after 触发器中
  新事件立即递归执行（通过 eventCollector）
  保证触发器的响应是同步的

在效果函数中
  新事件插入到当前事务队列（可指定位置）
  按队列顺序异步执行

这是触发触发器的唯一方法，请始终通过过程事件与实体的触发器交互。

## 核心方法

### trigger(when, triggerLevel)

触发该事件的触发器

参数：
  when: "before" | "after"  // 触发时机
  triggerLevel: number      // 触发等级

行为：
  调用 source.makeEvent(when, key, event, null, triggerLevel)
  调用 medium.viaEvent(when, key, event, null, triggerLevel)
  调用 target.takeEvent(when, key, event, null, triggerLevel)

### execute()

执行该事件（由 Transaction 调用）

执行顺序：
  调用 onExecute 回调
  执行 effects 中的所有效果
  调用 onComplete 回调

### spawnEvent(event)

标记另一个事件为该事件的子事件

行为：
  设置父子关系（用于日志嵌套）
  继承 simulate 标记
  共享事务收集器

## 核心函数

### doEvent(config, options?)

创建并发生一个过程事件，这是创建事件的主要方式

```typescript
doEvent({
    key: string                    // 事件key
    source: EventParticipant       // 事件来源
    medium: EventParticipant       // 事件媒介
    target: EventParticipant | EventParticipant[]  // 事件目标
    info?: Record<string,any>      // 事件信息
    effectUnits?: EffectUnit[]     // 效果单元数组
    doWhat?: () => void            // 事件执行时的回调
    onComplete?: (event) => void   // 事件完成后的回调
}, {
    position?: number | "top" | "bottom"  // 插入位置（仅在效果函数中有效）
})
```

收集行为：

在 before/after 触发器中
  添加到 eventCollector（立即递归执行）

在效果函数中
  插入到当前事务的指定位置
    "top"：队首（默认，优先执行）
    "bottom"：队尾（最后执行）
    数字：指定索引位置

没有当前事务
  创建新事务并加入事务队列

返回值：
  返回创建的 ActionEvent 对象

### 使用示例

```typescript
// 造成伤害事件
doEvent({
    key: "damage",
    source: enemy,
    medium: attack,
    target: player,
    effectUnits: [{
        key: "damage",
        params: { value: 10 }
    }]
})

// 治疗事件
doEvent({
    key: "heal",
    source: relic,
    medium: relic,
    target: player,
    effectUnits: [{
        key: "heal",
        params: { value: 5 }
    }]
})

// 抽卡事件
doEvent({
    key: "drawCard",
    source: player,
    medium: player,
    target: player,
    effectUnits: [{
        key: "drawFromDrawPile",
        params: { value: 3 }
    }]
})
```

## 模拟模式

模拟事件用于预测效果，不会实际修改游戏状态

特性：
  simulate 标记会自动继承给子事件
  模拟事件不会收集到事务队列
  触发器仍然会被触发（用于预测）
  效果函数中应检查 simulate 标记，避免实际修改状态

使用场景：
  预测伤害/治疗量
  检查条件是否满足
  AI 决策模拟
  UI 预览效果

创建模拟事件：

```typescript
const simulateEvent = new ActionEvent({
    key: "damage",
    source: enemy,
    medium: attack,
    target: player,
    effectUnits: [{ key: "damage", params: { value: 10 } }],
    simulate: true  // 标记为模拟事件
})
```

## 事件收集器 EventCollector

事件收集器用于在触发器中收集新事件，并立即递归执行

工作流程：

Transaction.executeEvent 执行事件时：
  设置 eventCollector 为空数组
  触发 before/after 触发器
  触发器中调用 doEvent 会将新事件添加到 eventCollector
  清除 eventCollector
  递归执行收集到的所有新事件

相关函数：
  setEventCollector(collector)：设置当前收集器
  getEventCollector()：获取当前收集器
  clearEventCollector()：清除当前收集器

## 事件级联

触发器可以创建新事件，新事件又可以触发其他触发器，形成事件级联

示例流程：

玩家使用攻击卡牌
  → 触发 "useCard" 事件
  → 触发器创建 "damage" 事件
  → 触发 "damage" 事件的 before 触发器
  → 遗物增加伤害
  → 执行 damage 效果
  → 触发 "damage" 事件的 after 触发器
  → 器官触发额外效果

级联控制：

使用 triggerLevel 控制触发器优先级
  level 越大越先触发
  同 level 按添加顺序触发

使用 position 控制事件插入位置
  "top"：优先执行
  "bottom"：延后执行

## 注意事项

事件执行是异步的
  所有事件执行方法都是 async
  需要使用 await 等待事件完成

触发器中的事件立即执行
  before/after 触发器中创建的事件会立即递归执行
  这保证了触发器的响应是同步的

效果函数中的事件插入队列
  效果函数中创建的事件会插入到当前事务队列
  可以通过 position 参数控制插入位置
  默认插入队首（优先执行）

避免无限递归
  触发器中创建的事件可能再次触发同一触发器
  需要在触发器中添加条件判断，避免无限循环
  使用 condition 字段限制触发条件

模拟模式的限制
  模拟事件不会实际修改状态
  效果函数需要检查 event.simulate 标记
  模拟事件不会收集到事务队列

事件参与者的角色
  source/medium/target 的选择影响触发器的触发
  合理设置参与者角色，确保触发器正确响应

## 常见事件 key

战斗相关：
  damage: 造成伤害
  heal: 治疗
  block: 获得格挡
  turnStart: 回合开始
  turnEnd: 回合结束
  battleStart: 战斗开始
  battleEnd: 战斗结束

卡牌相关：
  drawCard: 抽卡
  useCard: 使用卡牌
  discardCard: 弃牌
  exhaustCard: 消耗卡牌
  addCardToHand: 添加卡牌到手牌

状态相关：
  addStatus: 添加状态
  removeStatus: 移除状态
  gainEnergy: 获得能量
  loseEnergy: 失去能量

其他：
  death: 死亡
  gainGold: 获得金币
  gainRelic: 获得遗物

# 器官质量系统与操作方法

器官系统的完整功能，包括质量系统、售卖、修复、升级等操作。

## 质量系统 Mass System

器官可以拥有质量属性，质量归0时器官会损坏。

### 配置方式

在器官定义中添加质量相关属性：

```typescript
{
    label: "心脏",
    key: "original_organ_00001",
    quality: OrganQuality.Common,
    part: OrganPart.Heart,
    status: {
        "max-mass": 30  // 最大质量
    },
    current: ["mass"],  // 当前质量
    // ...
}
```

### 质量伤害

器官可以被攻击并受到质量伤害。使用 `damageOrgan` effect：

```typescript
doEvent({
    key: "attackOrgan",
    source: enemy,
    target: organ,
    effectUnits: [{
        key: "damageOrgan",
        params: {
            organ: organ,
            value: 10  // 造成10点质量伤害
        }
    }]
})
```

### 质量恢复

使用 `healOrgan` effect 恢复器官质量：

```typescript
doEvent({
    key: "healOrgan",
    source: player,
    target: organ,
    effectUnits: [{
        key: "healOrgan",
        params: {
            organ: organ,
            value: 5  // 恢复5点质量
        }
    }]
})
```

### 自动损坏

当器官质量归0时，会自动触发损坏（`breakOrgan`）。

例外情况：
  拥有 `INDESTRUCTIBLE`（不朽）标签的器官不会因质量归0而损坏
  允许通过标签系统定义其他例外

### 相关文件

Effect 实现
  `src/core/effects/organ/organEffects.ts` - damageOrgan 和 healOrgan

Effect 注册
  `src/static/list/system/effectMap.ts` - 注册到 effect 映射表

## 器官操作方法

### 售卖器官 sellOrgan

将器官售卖获得金钱。

使用方式：

```typescript
import { sellOrgan } from "@/core/objects/target/Organ"

// 售卖器官
const success = sellOrgan(player, organ)
```

售价计算：
  基础价格 = 黑市价格（基于稀有度和等级）
  实际售价 = 基础价格 × 随机折扣（0.8 ~ 1.1）
  售卖后器官消失（调用 loseOrgan）

实现位置：
  `src/core/objects/system/modifier/OrganModifier.ts` - sellOrgan 方法
  `src/core/objects/target/Organ.ts` - 包装函数

### 修复器官 repairOrgan

消耗物质修复损坏的器官。

使用方式：

```typescript
import { repairOrgan } from "@/core/objects/target/Organ"

// 修复器官
const success = await repairOrgan(player, organ)
```

修复机制：
  消耗物质 = 吞噬获取量 × 60%
  完全恢复质量到最大值
  移除 broken 触发器，重新激活 work 触发器
  如果物质不足，修复失败

实现位置：
  `src/core/objects/system/modifier/OrganModifier.ts` - repairOrgan 方法
  `src/static/list/target/organQuality.ts` - calculateRepairCost 函数

### 升级器官 upgradeOrgan

消耗资源提升器官等级。

使用方式：

```typescript
import { upgradeOrgan } from "@/core/objects/target/Organ"

// 升级器官
const success = await upgradeOrgan(player, organ)
```

升级机制：
  优先消耗物质（使用稀有度计算的升级成本）
  物质不足时消耗生命值作为备选
  升级成本 = 吞噬获取量 × 110% ± 10% 波动
  提升器官等级
  执行每级通用效果（如果配置了 perLevel）
  检查并触发里程碑效果（如果达到 milestone）

自定义升级成本：

```typescript
{
    label: "特殊器官",
    quality: OrganQuality.Rare,
    upgrade: {
        cost: 50,  // 固定成本
        // 或者
        cost: (organ) => organ.level * 20,  // 动态成本
        perLevel: {
            effects: [{ key: "gainMaxHealth", params: { value: 2 } }]
        },
        milestones: [
            {
                level: 5,
                effects: [{ key: "gainRelic", params: { relicKey: "special_relic" } }]
            }
        ]
    }
}
```

实现位置：
  `src/core/objects/system/modifier/OrganModifier.ts` - upgradeOrgan 方法
  `src/static/list/target/organQuality.ts` - calculateUpgradeCost 函数

### 吞噬器官 devourOrgan

消耗器官获得物质。

使用方式：

```typescript
import { devourOrgan } from "@/core/objects/target/Organ"

// 吞噬器官
const success = devourOrgan(player, organ)
```

吞噬收益：
  获得物质 = 吞噬获取量 × 等级 + 稀有度加成（20%）
  器官消失（调用 loseOrgan）

实现位置：
  `src/core/objects/system/modifier/OrganModifier.ts` - devourOrgan 方法

### 同化器官 assimilateOrgan

在战斗结束时，从敌人的器官中选择一个未损坏的器官进行同化。

使用方式：

```typescript
import { assimilateOrgan } from "@/core/objects/target/Organ"

// 同化器官
const success = assimilateOrgan(player, enemyOrgan)
```

同化机制：
  消耗物质 = 吞噬获取量 × 20%
  器官等级重置为 1
  获得器官（调用 acquireOrgan）

实现位置：
  `src/core/objects/system/modifier/OrganModifier.ts` - assimilateOrgan 方法
  `src/static/list/target/organQuality.ts` - calculateAssimilateCost 函数

## 设计原则

质量系统是可选的
  不是所有器官都需要质量系统
  没有 max-mass 属性的器官不会受到质量伤害

总是允许例外
  通过标签系统（如 INDESTRUCTIBLE）定义特殊行为
  不要硬编码规则，保持系统灵活性

资源消耗优先级
  升级：优先物质，不足时生命
  修复：仅消耗物质
  同化：仅消耗物质

数值后期调整
  当前数值（60%、110%、20% 等）仅供参考
  重点是机制的实现
  数值平衡在后期进行

# 水池行动扩展系统

水池行动扩展系统允许通过遗物解锁水池房间中的新行动选项。玩家在水池中可以选择不同的行动来提升自己。

## 核心概念

水池行动 (PoolAction)
  定义：在水池房间中可以执行的操作
  内置行动：汲取（获得物质）、升级（升级器官）、染血（获得印记）
  扩展行动：锻炼（需要遗物解锁）、锻炼鸡鸡（需要遗物解锁）等

行动状态管理
  存储位置：Player.enabledPoolActions (Set<string>)
  启用方式：通过 effect 添加到玩家的行动集合
  禁用方式：通过 effect 从玩家的行动集合移除

行动注册表
  全局注册表：poolActionRegistry
  存储所有扩展行动的定义和处理器
  通过 enabled 函数动态判断行动是否对玩家可用

## 系统架构

### 类型定义

文件：`src/core/types/poolAction.ts`

```typescript
export interface PoolAction {
  key: string           // 唯一标识
  title: string         // 显示标题
  icon: string          // 图标
  description: string   // 描述文本
  priority: number      // 排序优先级（越高越靠前）
  enabled: boolean | ((player: Player, context: PoolActionContext) => boolean)
  repeatable?: boolean  // 是否可重复执行（默认 false）
}

export interface PoolActionContext {
  layer: number         // 当前层级
  absorbAmount: number  // 汲取物质数量
}

export type PoolActionHandler = (
  player: Player,
  context: PoolActionContext
) => Promise<boolean> | boolean
```

### 注册表

文件：`src/static/registry/poolActionRegistry.ts`

主要方法：
  registerAction(config, handler) - 注册新行动
  getAvailableActions(player, context) - 获取玩家可用的行动列表
  executeAction(key, player, context) - 执行指定行动

工作流程：
  1. 应用启动时注册所有扩展行动
  2. PoolRoom 创建选项时调用 getAvailableActions()
  3. enabled 函数检查 player.enabledPoolActions 集合
  4. 返回按 priority 排序的可用行动列表

### 扩展行动

文件：`src/core/hooks/poolActions.ts`

锻炼行动 (exercise)
  优先级：70
  效果：消耗 30 物质，增加 5 最大生命
  可用条件：player.enabledPoolActions.has("exercise")
  可重复：true
  解锁方式：获得健身手环遗物


初始化：
  initPoolActions() 在应用启动时调用
  注册所有扩展行动到全局注册表

注意：
  内置行动（汲取、升级、染血）在 PoolRoom 中直接实现
  不需要通过注册表，始终可用

### 效果函数

文件：`src/core/effects/pool/poolActionEffects.ts`

启用行动效果 (enablePoolAction)
  参数：actionKey (string)
  效果：player.enabledPoolActions.add(actionKey)
  用途：遗物 possess 时启用新行动

禁用行动效果 (disablePoolAction)
  参数：actionKey (string)
  效果：player.enabledPoolActions.delete(actionKey)
  用途：遗物 lose 时禁用行动

注册到 effectMap：
  key: "enablePoolAction"
  key: "disablePoolAction"

## PoolRoom 集成

文件：`src/core/objects/room/PoolRoom.ts`

createChoices() 修改：
  1. 创建内置选项（汲取、升级、染血）
  2. 调用 poolActionRegistry.getAvailableActions() 获取扩展行动
  3. 为每个扩展行动创建 Choice 对象
  4. 点击时调用 poolActionRegistry.executeAction()

关键代码：
```typescript
const extendedActions = poolActionRegistry.getAvailableActions(nowPlayer, {
  layer: this.layer,
  absorbAmount: this.absorbAmount
})

for (const action of extendedActions) {
  choices.push(new Choice({
    title: action.title,
    description: action.description,
    icon: action.icon,
    onSelect: async () => {
      await poolActionRegistry.executeAction(
        action.key,
        nowPlayer,
        { layer: this.layer, absorbAmount: this.absorbAmount }
      )
    }
  }))
}
```

## 遗物扩展机制

### 健身手环示例

文件：`src/static/list/item/relicList.ts`

```typescript
{
  label: "健身手环",
  describe: ["水池中可以选择锻炼，消耗物质增加最大生命"],
  key: "original_relic_exercise",
  interaction: {
    possess: {
      target: { key: "owner" },
      effects: [{
        key: "enablePoolAction",
        params: { actionKey: "exercise" }
      }]
    },
    lose: {
      target: { key: "owner" },
      effects: [{
        key: "disablePoolAction",
        params: { actionKey: "exercise" }
      }]
    }
  }
}
```

工作流程：
  获得遗物 → possess 交互触发 → enablePoolAction effect
  → player.enabledPoolActions.add("exercise")
  → 锻炼选项出现在水池中

  失去遗物 → lose 交互触发 → disablePoolAction effect
  → player.enabledPoolActions.delete("exercise")
  → 锻炼选项消失

### 添加新行动的步骤

1. 在 poolActions.ts 中定义行动处理器

```typescript
export const meditateAction: PoolActionHandler = async (player, context) => {
  // 自定义逻辑：冥想恢复生命值
  const healAmount = 10
  const { doEvent } = await import("@/core/objects/system/ActionEvent")

  await doEvent({
    key: "heal",
    source: player,
    medium: player,
    target: player,
    effectUnits: [{
      key: "heal",
      params: { value: healAmount }
    }]
  })

  return true
}
```

2. 在 initPoolActions() 中注册行动

```typescript
poolActionRegistry.registerAction({
  key: "meditate",
  title: "冥想",
  icon: "🧘",
  description: "恢复 10 点生命值",
  priority: 60,
  repeatable: false,
  enabled: (player, context) => {
    return player.enabledPoolActions.has("meditate")
  }
}, meditateAction)
```

3. 创建解锁该行动的遗物

```typescript
{
  label: "禅修念珠",
  describe: ["水池中可以选择冥想，恢复生命值"],
  key: "original_relic_meditate",
  interaction: {
    possess: {
      target: { key: "owner" },
      effects: [{
        key: "enablePoolAction",
        params: { actionKey: "meditate" }
      }]
    },
    lose: {
      target: { key: "owner" },
      effects: [{
        key: "disablePoolAction",
        params: { actionKey: "meditate" }
      }]
    }
  }
}
```

## 设计优势

状态明确
  玩家的可用行动存储在 Player 对象中
  不依赖全局状态或遗物查询

自动清理
  失去遗物时自动禁用行动
  通过 lose 交互实现，无需手动管理

数据驱动
  遗物配置完全通过 TriggerMap
  无需编写代码，只需配置 JSON

易于扩展
  添加新行动只需三步：定义处理器、注册行动、创建遗物
  Mod 开发者可以轻松添加自定义行动

架构一致
  与器官奖励动作系统保持一致
  使用相同的 effect 系统和 TriggerMap 机制

内置行动分离
  内置行动在 PoolRoom 中直接实现
  扩展行动通过注册表管理
  避免注册表过于臃肿

## 注意事项

行动优先级
  priority 越高，在选项列表中越靠前显示
  建议：扩展行动 70 以下，为未来内置行动预留空间

enabled 函数性能
  每次创建选项时都会调用 enabled 函数
  保持函数简单，避免复杂计算

可重复行动
  repeatable: true 的行动可以多次执行
  目前实现中，执行后不会自动刷新选项
  需要玩家重新进入水池或手动刷新

行动执行失败
  executeAction 返回 boolean 表示成功/失败
  失败时会记录错误日志
  不影响其他行动的执行

内置行动不可禁用
  汲取、升级、染血是内置行动
  不通过注册表管理，无法被禁用
  如需禁用，需要修改 PoolRoom 代码

上下文信息
  PoolActionContext 提供层级和汲取数量信息
  可用于实现层级相关的行动效果
  例如：高层级时锻炼效果更好

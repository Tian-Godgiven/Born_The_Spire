# 条件系统 Condition System

提供字符串语法的条件检查功能，用于判断游戏状态是否满足要求

## 分层架构

条件系统采用三层设计，每层职责清晰：

### 第一层：目标解析 (TargetSpec)

将目标标识符解析为实际的游戏对象

```
"owner"         → 玩家实体
"source"        → 事件来源
"allEnemies"    → 所有敌人数组
"enemy"         → 单个敌人（事件目标）
"allEnemies.random"  → 随机敌人
"allEnemies.first"   → 第一个敌人
"allEnemies.last"    → 最后一个敌人
```

### 第二层：属性访问 (EntityAccessor)

从实体对象上读取属性值

```
status(health)     → 读取 entity.status 中的属性值
current(energy)    → 读取 entity.current 中的当前值
hasStatus(bleed)   → 是否有流血状态（值 > 0）
hasOrgan(heart)    → 是否拥有心脏器官
hasRelic(foo)      → 是否拥有 foo 遗物
hasCard(strike)    → 手牌中是否有打击牌
healthPercent()    → 健康百分比（当前生命/最大生命）
```

### 第三层：条件评估 (ConditionSystem)

组合目标和属性进行布尔判断

```
"owner.status(health) > 100"
"allEnemies.any.hasOrgan(heart)"
"source.current(energy) <= 2"
"battle.turn >= 5"
```

## 基础语法

### 比较操作符

支持以下操作符：

```
==   等于
!=   不等于
>    大于
>=   大于等于
<    小于
<=   小于等于
```

### 字符串条件

单个条件采用 `目标.属性 操作符 值` 格式：

```
"owner.status(health) > 100"
"source.current(energy) <= 2"
"battle.turn == 3"
"scene == combat"
```

### AND 组合

数组形式默认是 AND 关系（所有条件都要满足）：

```typescript
[
    "owner.status(health) > 50",
    "battle.turn >= 3"
]
// 玩家生命 > 50  且  回合数 >= 3
```

### OR 组合

使用 `or` 字段：

```typescript
{
    or: [
        "owner.status(health) < 30",
        "source.hasStatus(bleed)"
    ]
}
// 玩家生命 < 30  或  来源有流血
```

### NOT 组合

使用 `not` 字段：

```typescript
{
    not: "battle.turn == 1"
}
// 不是第一回合
```

### 混合组合

可以嵌套使用 and/or/not：

```typescript
{
    and: [
        "owner.status(health) > 50",
        {
            or: [
                "source.hasStatus(power)",
                "source.hasStatus(ritual)"
            ]
        }
    ]
}
// 生命 > 50  且 ( 有力量 或 有仪式 )
```

## 目标修饰符

### 修饰符语法

使用点语法对目标数组进行修饰：

```
allEnemies.random   → 从敌人数组中随机选一个
allEnemies.first    → 选择第一个敌人
allEnemies.last     → 选择最后一个敌人
```

### 修饰符使用场景

```
// 直接目标（单个实体）
"owner.status(health) > 100"  ✓

// 数组目标需要修饰符
"allEnemies.status(health) > 100"  ✗ 错误！allEnemies 是数组
"allEnemies.random.status(health) > 100"  ✓ 正确！

// 量词修饰符（any/all）用于集合判断
"allEnemies.any.hasOrgan(heart)"  → 任意敌人有心脏
"allEnemies.all.hasOrgan(heart)"  → 所有敌人都有心脏
```

## 集合量词

### .any 修饰符

判断集合中是否任意一个元素满足条件：

```
"allEnemies.any.hasOrgan(heart)"   → 任意敌人有心脏
"allEnemies.any.status(health) < 50"  → 任意敌人生命 < 50
```

### .all 修饰符

判断集合中是否所有元素都满足条件：

```
"allEnemies.all.hasOrgan(heart)"   → 所有敌人都有心脏
"allEnemies.all.status(health) < 50"  → 所有敌人都生命 < 50
```

## 使用示例

### 示例1：血量检查

```typescript
// 玩家生命大于50
"owner.current(health) > 50"

// 玩家生命低于30
"owner.current(health) < 30"

// 玩家生命低于50%（使用百分比）
"owner.healthPercent() <= 0.5"

// 任意敌人生命低于20%
"allEnemies.any.current(health) < 20"
```

### 示例2：回合检查

```
// 当前是战斗中
"scene == combat"

// 回合数大于等于3
"battle.turn >= 3"

// 是第一回合
"battle.turn == 1"
```

### 示例3：器官/遗物检查

```
// 玩家拥有心脏器官
"owner.hasOrgan(heart)"

// 任意敌人有再生器官
"allEnemies.any.hasOrgan(regeneration)"

// 玩家拥有吸血遗物
"owner.hasRelic(vampire_relic)"
```

### 示例4：组合条件

```typescript
// 玩家生命低于50% 且 回合数大于3
[
    "owner.healthPercent() <= 0.5",
    "battle.turn > 3"
]

// 回合数是偶数 或 玩家有能量
{
    or: [
        "battle.turn % 2 == 0",
        "owner.current(energy) > 0"
    ]
}

// 所有敌人都有防御 且 玩家生命满格
{
    and: [
        "allEnemies.all.hasStatus(armor)",
        "owner.current(health) == owner.status(maxHealth)"
    ]
}
```

### 示例5：敌人行为条件

```
// 敌人血量低于30%时狂暴
{ selfHealth: { below: 30 } }

// 第1回合强化
{ turn: { equals: 1 } }

// 每3回合行动一次
{ turn: { mod: [3, 0] } }

// 血量高于66%防守，33%-66%攻击，低于33%狂暴
// 第一阶段：health > 66%
// 第二阶段：health > 33%
// 第三阶段：health < 33%
```

## 注意事项

### 1. 百分比比较

健康百分比使用 `healthPercent()` 访问器：

```
// 玩家生命低于50%
"owner.healthPercent() <= 0.5"

// 玩家生命高于80%
"owner.healthPercent() > 0.8"
```

### 2. 字符串 vs 对象语法

简单条件用字符串，复杂条件用对象：

```
// 简单条件 → 字符串
"owner.status(health) > 50"

// AND → 数组
["owner.status(health) > 50", "battle.turn > 3"]

// OR/NOT → 对象
{ or: ["cond1", "cond2"] }
{ not: "cond1" }
```

### 3. 常见错误

```
// 错误：数组目标没有使用修饰符
"allEnemies.status(health) > 100"  ✗

// 错误：缺少操作符
"owner.status(health) 100"  ✗

// 错误：无效的访问器
"owner.status(health点)"  ✗
```

### 4. 自定义访问器

Mod作者可以注册自定义访问器：

```typescript
import { registerAccessor } from "@/core/types/EntityAccessor"

registerAccessor("myMod_customValue", (arg, entity) => {
    return (entity as any).myCustomField ?? 0
})

// 使用： "owner.myMod_customValue(someArg) > 10"
```

## 迁移指南

### 旧条件语法（已废弃）

```typescript
// 旧格式：object 语法
condition: { sourceStatus: { key: "cooldown", value: 0, op: "lte" } }
condition: { ownerHealthPercent: { value: 0.5, op: "lte" } }
```

### 新条件语法

```typescript
// 新格式：字符串表达式
"source.status(cooldown) <= 0"
"owner.current(health) <= 50"
"owner.healthPercent() <= 0.5"

// 或使用对象语法（复杂条件）
{
    and: [
        "owner.current(health) < 50",
        "battle.turn >= 3"
    ]
}
```

### 迁移示例

```typescript
// 旧：血量低于50%时触发
{
    condition: { ownerHealthPercent: { value: 0.5, op: "lte" } }
}

// 新：血量低于50%时触发
{
    condition: "owner.healthPercent() <= 0.5"
}

// 旧：冷却为0时触发
{
    condition: { sourceStatus: { key: "cooldown", value: 0, op: "lte" } }
}

// 新：冷却为0时触发
{
    condition: "source.status(cooldown) == 0"
}

// 旧：血量高于66%时防守
{
    condition: { selfHealth: { above: 66 } }
}

// 新：血量高于66%时防守（需要根据实际数据结构调整）
{
    condition: "self.healthPercent() > 0.66"
}
```

## 支持的目标类型

```
目标类型                     说明
------------------------------------------------------------------------------
item                        物品本身（遗物/器官/卡牌）
owner                       物品拥有者（玩家/敌人）
source                      事件来源
target                      事件目标（或目标数组）
battle                      当前战斗对象
player                      玩家角色（从战斗中获取）
enemy                       敌人角色（从事件目标中获取）
allEnemies                  所有存活敌人
allAllies                   所有盟友
allEntities                 所有实体（玩家+敌人）
turnNumber                  当前回合数
```

```
事件类型                     说明

turnStart                   回合开始
turnEnd                     回合结束
battleStart                 战斗开始
battleEnd                   战斗结束
gainPower                   获得力量
applyState                  应用状态
```
